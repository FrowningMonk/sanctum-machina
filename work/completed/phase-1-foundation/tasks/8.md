# Task 8: ModelManagerScreen + ModelManagerViewModel

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Собрать первый экран приложения — `ModelManagerScreen` поверх `ModelRegistry` (Task 6) и `strings.xml` (Task 7). Экран отображает список локальных LLM-моделей, позволяет скачать/отменить/загрузить каждую из них и служит entry point всей Phase 1 (из него пользователь уходит в чат через NavHost, который соберёт Task 10).

`ModelManagerViewModel` — тонкий `@HiltViewModel`, проксирующий `registry.models: StateFlow<List<ModelEntry>>` в UI и делегирующий действия обратно в registry. ViewModel не знает про `NavController` — навигационные намерения эмитятся через `SharedFlow<NavEvent>`, который подписывает родитель (NavHost в Task 10).

UI строится по Material 3 в минималистичном стиле `ux-guidelines.md`: Scaffold + TopAppBar + LazyColumn + Card per model, все строки — только из `strings.xml` (экран его читает, но не модифицирует — этим владеет Task 7). Статус-бейдж и action-кнопка внутри карточки детерминированы полями `ModelEntry.downloadStatus` / `initStatus`, что делает экран чистой проекцией состояния registry.

## What to do

1. Создать `ModelManagerViewModel` (`@HiltViewModel`, `@Inject constructor(registry: ModelRegistry)`):
   - Публичный `val models: StateFlow<List<ModelEntry>> = registry.models` (прямая проекция, без промежуточного `stateIn` — registry сам держит `StateFlow`).
   - Публичный `val navEvents: SharedFlow<NavEvent>` (замкнутый `MutableSharedFlow<NavEvent>(extraBufferCapacity = 1)`), где `NavEvent` — sealed class / interface с вариантом `OpenChat(val modelName: String)`.
   - `fun onDownload(entry: ModelEntry)` — делегирует `registry.download(entry.model)` (канонический метод из Task 6, возвращает `Flow<ModelDownloadStatus>`). Поскольку registry сам обновляет `models` StateFlow через DownloadRepository-колбэки, ViewModel может либо проигнорировать Flow (fire-and-forget), либо сделать `registry.download(entry.model).launchIn(viewModelScope)` для явной подписки. Оба варианта допустимы; выбираем fire-and-forget для простоты.
   - `fun onCancel(modelName: String)` — делегирует `registry.cancelDownload(modelName)`.
   - `fun onLoad(modelName: String)` — `viewModelScope.launch { navEvents.emit(NavEvent.OpenChat(modelName)) }`. Никакой работы с `NavController` внутри VM.

2. Создать `ModelManagerScreen` (`@Composable fun ModelManagerScreen(onLoad: (String) -> Unit, viewModel: ModelManagerViewModel = hiltViewModel())`):
   - Scaffold с TopAppBar, заголовок — `stringResource(R.string.model_manager_title)`.
   - `val models by viewModel.models.collectAsStateWithLifecycle()`.
   - `LaunchedEffect(Unit) { viewModel.navEvents.collect { event -> when (event) { is NavEvent.OpenChat -> onLoad(event.modelName) } } }`.
   - LazyColumn (обычный порядок сверху вниз) с `items(models, key = { it.model.name })`, каждая — `ModelCard(entry = it, onDownload, onCancel, onLoad)`.
   - ModelCard (работаем с `entry: ModelEntry`, где `entry.model: Model`, `entry.downloadStatus: ModelDownloadStatus`, `entry.initStatus: ModelInitStatus`):
     - Material3 `Card` с отступом между карточками.
     - Headline: `entry.model.name`.
     - Subline: размер в ГБ через готовый ключ `model_size_gb_format` с плейсхолдером `%1$.1f` (Float): `stringResource(R.string.model_size_gb_format, entry.model.sizeInBytes / 1_073_741_824.0f)`.
     - Статус-бейдж + action-кнопка строятся по правилам ниже.
   - Статусная матрица (все подписи — из `strings.xml`, ключи канонические из Task 7):
     - `entry.downloadStatus.status == NOT_DOWNLOADED` → badge `model_status_not_downloaded`, button `btn_download` → `onDownload(entry)`.
     - `entry.downloadStatus.status == IN_PROGRESS` → badge `model_status_downloading`, `LinearProgressIndicator(progress = { (entry.downloadStatus.receivedBytes.toFloat() / entry.downloadStatus.totalBytes.coerceAtLeast(1)).coerceIn(0f, 1f) })` + текст прогресса по ключу `model_download_progress_format` с двумя аргументами Int MB: `stringResource(R.string.model_download_progress_format, (entry.downloadStatus.receivedBytes / 1_048_576L).toInt(), (entry.downloadStatus.totalBytes / 1_048_576L).toInt())`; button `btn_cancel` → `onCancel(entry.model.name)`. Если `entry.downloadStatus.totalBytes == 0L` — показать indeterminate `LinearProgressIndicator()` и скрыть текст прогресса.
     - `entry.downloadStatus.status == SUCCEEDED` → badge `model_status_downloaded`, button `btn_load` (enabled) → `onLoad(entry.model.name)`.
     - `entry.downloadStatus.status == FAILED` → badge `model_status_failed` + строка ошибки через `stringResource(R.string.model_error_prefix, entry.downloadStatus.errorMessage ?: "")`, button `btn_retry` → `onDownload(entry)`.
   - Кнопка Load включена **только** в ветке SUCCEEDED (во всех остальных состояниях она либо отсутствует, либо заменена кнопкой Download/Cancel/Retry — `enabled = true` для присутствующей ветки).
   - Байты → МБ для прогресса: `bytes / 1_048_576` целочисленно, формат через `stringResource` с позиционными аргументами `%1$d` / `%2$d`.

3. Визуальный стиль — строго по `ux-guidelines.md` (минимализм, Material 3, без декоративных иконок в MVP, цвета из темы). Тон русский, строки только из `strings.xml`.

4. Никаких изменений в `strings.xml`. Если обнаружится отсутствующий ключ — остановиться и сообщить: правит Task 7, не текущая задача.

5. Проверить, что все используемые `R.string.*` ключи реально существуют в `app/src/main/res/values/strings.xml` (читаем, не модифицируем).

## Details

**Files:**
- `app/src/main/kotlin/app/sanctum/machina/ui/modelmanager/ModelManagerViewModel.kt` — новый файл. `@HiltViewModel class ModelManagerViewModel @Inject constructor(private val registry: ModelRegistry) : ViewModel()`. Держит `MutableSharedFlow<NavEvent>` (private) + read-only `SharedFlow` наружу. `NavEvent` sealed-класс определяется рядом (в этом же пакете) — `sealed interface NavEvent { data class OpenChat(val modelName: String) : NavEvent }`.
- `app/src/main/kotlin/app/sanctum/machina/ui/modelmanager/ModelManagerScreen.kt` — новый файл. Публичный Composable `ModelManagerScreen(onLoad: (String) -> Unit, viewModel: ModelManagerViewModel = hiltViewModel())`. Внутренние приватные Composable'ы: `ModelList`, `ModelCard`, `ModelStatusRow` (по желанию, чтобы не раздувать один метод).

**Dependencies:**
- Task 6 (ModelRegistry + DI binding + `ModelEntry` / enum статусов).
- Task 7 (strings.xml, Hilt Application, MainActivity, тема). Все нужные ключи строк уже должны быть в `strings.xml`.
- Библиотеки (уже должны быть подключены в `:app` через Task 7): `androidx.hilt:hilt-navigation-compose`, `androidx.lifecycle:lifecycle-runtime-compose` (для `collectAsStateWithLifecycle`), Compose Material3 из BOM 2026.03.00.

**Edge cases:**
- `totalBytes == 0L` в IN_PROGRESS (ещё не получили HTTP Content-Length) → indeterminate `LinearProgressIndicator()`, текст прогресса показывать только когда `totalBytes > 0`.
- Пустой список моделей (registry инициализирован, но `models` = emptyList) — для Phase 1 не реалистичен (registry строится из зашитых allowlisted моделей), но UI не должен падать: LazyColumn просто пустой.
- Очень длинное `entry.downloadStatus.errorMessage` в FAILED — `Text` с `maxLines = 3`, `overflow = TextOverflow.Ellipsis`, чтобы карточка не раздувалась.
- Быстрые клики по «Скачать» / «Отмена» — методы ViewModel идемпотентны за счёт делегирования registry; UI не блокирует кнопки искусственно (это ответственность registry).
- Повторный вход на экран (back из чата) — `StateFlow` даёт текущее состояние мгновенно, без «мигания» NOT_DOWNLOADED поверх уже SUCCEEDED.

**Implementation hints:**
- LazyColumn — обычный порядок (`reverseLayout = false` по умолчанию), первая модель сверху.
- Прогресс для `LinearProgressIndicator`: `progress = { (entry.downloadStatus.receivedBytes.toFloat() / entry.downloadStatus.totalBytes.coerceAtLeast(1)).coerceIn(0f, 1f) }` (лямбда-форма из Material3 1.2+ избегает анимационных артефактов при частых обновлениях; `coerceAtLeast(1)` защищает от деления на ноль, отдельная ветка `totalBytes == 0L` всё равно нужна, чтобы показать indeterminate).
- `collectAsStateWithLifecycle()` предпочтительнее `collectAsState()` — корректно останавливает сбор при уходе экрана в фон.
- `NavEvent` эмитить именно через `SharedFlow` (не `StateFlow`): событие одноразовое, StateFlow его «залипит» и повторит после ротации.
- Форматирование размера: ключ `model_size_gb_format` с аргументом `%1$.1f` (**Float**, не Double — иначе Android формат-парсер бросит `IllegalFormatConversionException`); в коде — `stringResource(R.string.model_size_gb_format, entry.model.sizeInBytes / 1_073_741_824.0f)`.
- Форматирование прогресса: ключ `model_download_progress_format` с двумя `%1$d` / `%2$d` (Int MB), получаем через `(bytes / 1_048_576L).toInt()`.
- Префикс ошибки: ключ `model_error_prefix` с `%1$s` — берём `entry.downloadStatus.errorMessage ?: ""` (nullable-safe).
- Не тестируем Compose UI (см. patterns.md § Test Infrastructure и user-spec D8) — TDD Anchor в задаче отсутствует намеренно; верификация выполняется руками на устройстве.
- Decision D10 (минималистичный Material 3) — никаких градиентов/лишних иконок; бейдж статуса — просто `Text` с `MaterialTheme.colorScheme` для FAILED-ветки (error color).
