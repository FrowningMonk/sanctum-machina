# Task 0: Design System

## Required Skills

Перед выполнением задачи загрузи:

## Description

Создать визуальный фундамент Sanctum Machina на основе спецификации `design_handoff_phase3_ui`. Текущий `Theme.kt` использует Material 3 динамический цвет (Android 12+) и дефолтный `lightColorScheme`/`darkColorScheme` — никакой корпоративной айдентики нет. Нужно полностью заменить это на Sanctum-токены: 13 токенов для светлой темы, 13 для тёмной, все задокументированы в `sanctum-tokens.jsx`.

Sanctum Machina намеренно **не является стандартным Material**: `borderRadius 2dp` вместо 12, hairline-бордеры вместо surface elevation, serif-шрифт для заголовков. Всё это должно быть прошито в тему так, чтобы задачи 7–11 просто использовали токены из `MaterialTheme.colorScheme` и `MaterialTheme.typography`, без разрозненных хардкодов в каждом экране.

Три центральных результата задачи:
1. `SanctumColors` + `SanctumTypography` + global shapes → `MaterialTheme` видит их через override `colorScheme`/`typography`/`shapes`.
2. `SanctumIncognitoTheme` — отдельный composable-враппер с тёмной Sanctum-палитрой, независимо от системной темы. Применяется к ветке `chat/quick` в `SanctumApp.kt`.
3. `SanctumIcons.kt` с 21 кастомным `ImageVector` + `SmSigil` composable. После этой задачи все последующие UI-задачи берут иконки только из `SanctumIcons`, не из `Icons.Outlined`.

## What to do

1. Прочитать токены из файлов `design_handoff_phase3_ui/sanctum-tokens.jsx`, `sanctum-atoms.jsx`, `README.md` перед написанием любого кода. Эти файлы — источник правды для цветов, SVG-путей иконок и сигила.

2. Создать `SanctumColors.kt` — data class с 13 полями `Color` для каждой темы. Определить два singleton-объекта `SanctumLightColors` и `SanctumDarkColors` с hex- и oklch-значениями из токенов. oklch-цвета (`accent`, `danger`, `incognitoEdge`) конвертировать в `Color(r, g, b)`. Завести `LocalSanctumColors` через `compositionLocalOf` и extension property `MaterialTheme.sanctumColors`.

3. Изменить `Theme.kt` — убрать dynamic color и дефолтные схемы. Добавить параметр `darkTheme: Boolean = isSystemInDarkTheme()`. Заменить body на маппинг `SanctumColors` → Material 3 `ColorScheme`: `background = colors.bg`, `surface = colors.bgRaised`, `surfaceVariant = colors.bgSunk`, `onBackground = colors.ink`, `onSurface = colors.ink`, `primary = colors.accent`, `onPrimary = colors.accentInk`, `error = colors.danger`. Добавить `CompositionLocalProvider(LocalSanctumColors provides colors)`. Добавить override `shapes`: `Shapes(extraSmall = RoundedCornerShape(2.dp), small = RoundedCornerShape(2.dp), medium = RoundedCornerShape(2.dp), large = RoundedCornerShape(2.dp), extraLarge = RoundedCornerShape(22.dp))`.

4. Создать `SanctumTypography.kt` — три font family через `androidx.compose.ui:ui-text-google-fonts`: Cormorant Garamond, Inter, JetBrains Mono. Определить `SanctumTypography: Typography` — `displayLarge`/`headlineLarge`/`headlineMedium`/`headlineSmall` используют Cormorant Garamond 500; `bodyLarge`/`bodyMedium`/`bodySmall`/`titleMedium` используют Inter; `labelSmall` использует JetBrains Mono. Размеры из README: display 40/26/22sp, body 15/14.5/13/12sp, kicker 10sp uppercase letter-spacing 0.18em.

5. Обновить `Color.kt` — убрать Gallery-заглушки (`Purple80` и т.д.), добавить именованные Sanctum-константы (`SanctumAccentLight`, `SanctumAccentDark`, `SanctumDangerLight`, и т.д.), на которые ссылается `SanctumColors.kt`.

6. Создать `SanctumIcons.kt` — 21 константа `ImageVector`. SVG-пути из `sanctum-tokens.jsx` (`paths`-объект). Имена: `IconMenu`, `IconSettings`, `IconPlus`, `IconBolt`, `IconGhost`, `IconEyeOff`, `IconSend`, `IconSend2`, `IconMic`, `IconAttach`, `IconStar`, `IconStarFill`, `IconMore`, `IconCheck`, `IconBack`, `IconEdit`, `IconTrash`, `IconWarn`, `IconChevronDown`, `IconDownload`, `IconSparkle`.

7. Создать `SmSigil.kt` в пакете `ui/components` — Composable `SmSigil(modifier: Modifier = Modifier, size: Dp = 20.dp, color: Color = LocalSanctumColors.current.accent)`. SVG из `sanctum-atoms.jsx`: внешний ромб (polygon 10,2 18,10 10,18 2,10), центральный круг r=2.2, верхняя спица (line 10,2 → 10,7.8), нижняя спица (line 10,12.2 → 10,18). Рисовать через Compose `Canvas` + `drawPath`/`drawCircle`. strokeWidth = 1.2dp, round cap/join.

8. Создать `SanctumIncognitoTheme.kt` — `@Composable fun SanctumIncognitoTheme(content: @Composable () -> Unit)`. Вызывает `SanctumTheme(darkTheme = true, content = content)`. Всегда применяет тёмную Sanctum-палитру независимо от системной темы.

9. Обновить `SanctumApp.kt` — обернуть весь `NavHost` в `SanctumTheme { ... }`. Добавить TODO-комментарий о применении `SanctumIncognitoTheme` к ветке `chat/quick` (которую Task 7 добавит).

10. Добавить зависимость `androidx.compose.ui:ui-text-google-fonts` в `app/build.gradle.kts` и `gradle/libs.versions.toml`.

## Details

**Theme.kt — текущее состояние и что менять:**
Файл сейчас — 24 строки. Использует `isSystemInDarkTheme()` + `Build.VERSION.SDK_INT >= S` guard для `dynamicDarkColorScheme`. Нужно полностью убрать ветку с `Build.VERSION.SDK_INT >= Build.VERSION_CODES.S` и заменить body на создание `ColorScheme` из `SanctumColors`. Добавить параметр `darkTheme: Boolean = isSystemInDarkTheme()` к `SanctumTheme`. Финальный вызов: `MaterialTheme(colorScheme = colorScheme, typography = SanctumTypography, shapes = SanctumShapes, content = content)`.

**Color.kt — что менять:**
Файл существует в пакете `ui/theme`. Скорее всего содержит Gallery-заглушки (`Purple80`, `Purple40` и т.д.) — удалить их все. Добавить `Color` константы для каждого hex-токена, чтобы `SanctumColors.kt` мог их импортировать: `val ParchmentBg = Color(0xFFF4EFE6)`, `val ParchmentBgSunk = Color(0xFFEBE4D6)`, и т.д.

**SanctumApp.kt — текущее состояние:**
41 строка, минимальный NavHost с тремя маршрутами (`model_manager`, `chat/{modelName}`, `about`). `SanctumTheme` нигде не применён — добавить вокруг `NavHost`. Маршрут `chat/quick` появится в Task 7, поэтому добавить только `TODO("Task 7: wrap chat/quick composable in SanctumIncognitoTheme")` как комментарий.

**oklch → Color конвертация (приближения для начала работы):**
- `accent` light: `oklch(0.58 0.14 40)` ≈ `Color(0xFFB5531A)`
- `accent` dark: `oklch(0.68 0.14 42)` ≈ `Color(0xFFCC6B2A)`
- `danger` light: `oklch(0.56 0.16 28)` ≈ `Color(0xFFAD3818)`
- `danger` dark: `oklch(0.66 0.17 28)` ≈ `Color(0xFFC44E28)`
- `incognitoEdge` light: `oklch(0.62 0.16 42)` ≈ `Color(0xFFBF5E22)`
- `incognitoEdge` dark: `oklch(0.68 0.16 42)` ≈ `Color(0xFFCC6B2E)`
Уточнить через CSS oklch-калькулятор при реализации; значения выше — стартовые приближения.

**Google Fonts зависимость:**
Добавить в `app/build.gradle.kts` в блок `dependencies`:
```
implementation(libs.androidx.compose.ui.text.google.fonts)
```
Добавить в `gradle/libs.versions.toml` в `[libraries]`:
```
androidx-compose-ui-text-google-fonts = { group = "androidx.compose.ui", name = "ui-text-google-fonts" }
```
(Версия через BOM — явный `version.ref` не нужен при использовании `platform(libs.androidx.compose.bom)`.)

**SanctumIcons — подход к ImageVector:**
Использовать `ImageVector.Builder`. Viewport 20×20. strokeWidth = 1.5f, StrokeCap.Round, StrokeJoin.Round. Stroke color при создании задать `SolidColor(Color.Black)` — при использовании через `Icon(imageVector, tint = ...)` он будет overridden. Для иконок с fill (например `starFill`, `more`, `warn`-dot) задать `fill = SolidColor(Color.Black)`.

**SmSigil — точные координаты из sanctum-atoms.jsx:**
- Внешний ромб: `polygon points="10 2 18 10 10 18 2 10"` → Path: `moveTo(10,2); lineTo(18,10); lineTo(10,18); lineTo(2,10); close()`
- Центральный круг: `circle cx="10" cy="10" r="2.2"` → `drawCircle(center, radius)`
- Верхняя спица: `line x1="10" y1="2" x2="10" y2="7.8"` → `drawLine`
- Нижняя спица: `line x1="10" y1="12.2" x2="10" y2="18"` → `drawLine`
Viewport 20×20, `strokeWidth = 1.2f`. В дизайне нет горизонтальных спиц — только верхняя и нижняя.

**Регрессия:**
Существующие экраны (`ModelManagerScreen`, `ChatScreen`, `AboutScreen`) не ломать. Они ссылаются на `MaterialTheme.colorScheme.*` и `MaterialTheme.typography.*` — после подмены схемы они получат Sanctum-токены автоматически, без изменений. Проверить компиляцию перед отправкой на review.
