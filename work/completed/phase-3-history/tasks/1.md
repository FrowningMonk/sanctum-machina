# Task 1: Room foundation

## Required Skills

Before starting, load:
- `/skill:code-writing` — [SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

This task establishes the entire Room data layer that every subsequent Phase 3 task depends on. There is no Phase 3 persistence without it.

The work has four parts: (1) add Room 2.7.x to the Gradle version catalog and app module dependencies; (2) create the v1 schema entities (`ChatEntity`, `MessageEntity`) and DAOs (`ChatDao`, `MessageDao`) with suspend/Flow-only APIs; (3) create `SanctumDatabase` with `exportSchema = true` pointing to `app/schemas/`; (4) write the instrumented DAO tests that verify CRUD and ON DELETE CASCADE behavior.

Key schema constraints from tech-spec Decision 1:
- `chats.project_id` is nullable INTEGER with **no FK constraint** — the `projects` table does not exist in v1. Phase 4 adds it via `Migration(1,2)`.
- `project_files` table is **NOT** created in v1. Any implementation that creates it deviates from spec.

The schema JSON at `app/schemas/app.sanctum.machina.data.SanctumDatabase/1.json` must be generated and committed to git (AC-R5). This file is produced automatically by KSP when `exportSchema = true` and the `schemaDirectory` annotation processor argument is set.

## What to do

1. Add Room `2.7.x` to `gradle/libs.versions.toml`: new `room` version entry; four library entries — `room-runtime`, `room-ktx`, `room-compiler`, `room-testing`.

2. In `app/build.gradle.kts`: add `implementation(libs.room.runtime)`, `implementation(libs.room.ktx)`, `ksp(libs.room.compiler)` in the `dependencies` block. Add `androidTestImplementation(libs.room.testing)`. Add the KSP schema export argument in `android { defaultConfig { ... } }`:
   ```
   ksp {
       arg("room.schemaLocation", "$projectDir/schemas")
   }
   ```
   Also add `androidTestImplementation` for `androidx-test-runner` and `junit` so the androidTest source set can compile.

3. Create `ChatEntity` at `app/src/main/kotlin/app/sanctum/machina/data/model/ChatEntity.kt` — `@Entity(tableName = "chats")` with all v1 columns: `id` (PK AUTOINCREMENT), `projectId` (nullable Long, no FK), `modelId` (TEXT NOT NULL), `title` (TEXT nullable), `isManuallyTitled` (Int NOT NULL DEFAULT 0), `createdAt` (Long NOT NULL), `lastMessageAt` (Long NOT NULL). Add index on `lastMessageAt`.

4. Create `MessageEntity` at `app/src/main/kotlin/app/sanctum/machina/data/model/MessageEntity.kt` — `@Entity(tableName = "messages", foreignKeys = [ForeignKey(entity = ChatEntity::class, parentColumns = ["id"], childColumns = ["chatId"], onDelete = CASCADE)], indices = [Index("chatId"), Index("createdAt")])` with all v1 columns: `id` (PK AUTOINCREMENT), `chatId` (Long NOT NULL), `role` (TEXT NOT NULL), `text` (TEXT NOT NULL DEFAULT ""), `thinkingText` (TEXT nullable), `imagePath` (TEXT nullable), `audioPath` (TEXT nullable), `createdAt` (Long NOT NULL), `tokenCount` (Int nullable).

5. Create `ChatDao` at `app/src/main/kotlin/app/sanctum/machina/data/dao/ChatDao.kt` — `@Dao` interface with: `suspend fun insert(chat: ChatEntity): Long`, `suspend fun update(chat: ChatEntity)`, `suspend fun deleteById(id: Long)`, `suspend fun getById(id: Long): ChatEntity?`, `fun observeAll(): Flow<List<ChatEntity>>` (ordered by `lastMessageAt DESC`).

6. Create `MessageDao` at `app/src/main/kotlin/app/sanctum/machina/data/dao/MessageDao.kt` — `@Dao` interface with: `suspend fun insert(message: MessageEntity): Long`, `suspend fun deleteById(id: Long)`, `suspend fun getByChatId(chatId: Long): List<MessageEntity>`, `fun observeByChat(chatId: Long): Flow<List<MessageEntity>>` (ordered by `createdAt ASC`), `suspend fun countByChatId(chatId: Long): Int`.

7. Create `SanctumDatabase` at `app/src/main/kotlin/app/sanctum/machina/data/SanctumDatabase.kt` — `@Database(entities = [ChatEntity::class, MessageEntity::class], version = 1, exportSchema = true)` abstract class extending `RoomDatabase`. Two abstract DAO accessors. Companion object with `fun create(context: Context): SanctumDatabase` that calls `Room.databaseBuilder(context, SanctumDatabase::class.java, "sanctum.db").build()`. Enable foreign keys support via `addCallback` that executes `PRAGMA foreign_keys = ON` on open.

8. Create `app/src/androidTest/kotlin/app/sanctum/machina/data/dao/ChatDaoTest.kt` — instrumented test using `Room.inMemoryDatabaseBuilder`. Cover: insert and retrieve by id, update title, delete by id, `observeAll` Flow emits on insert.

9. Create `app/src/androidTest/kotlin/app/sanctum/machina/data/dao/MessageDaoTest.kt` — instrumented test. Cover: insert and retrieve by chatId, `observeByChat` sorted by `createdAt`, ON DELETE CASCADE (delete chat row → message rows gone), `countByChatId` returns 0 after cascade.

10. Create `app/src/androidTest/kotlin/app/sanctum/machina/data/SanctumDatabaseTest.kt` — instrumented test. Cover: fresh DB opens without error, FK enforcement is on (try inserting a message with a non-existent `chatId` → expect SQLiteConstraintException), schema has both `chats` and `messages` tables (query `sqlite_master`), no `project_files` table exists.

**Entity implementation notes:**

`ChatEntity` — use `@ColumnInfo(name = "project_id")` on `projectId: Long?`. Explicitly do NOT add a `@ForeignKey` for `projectId` — the `projects` table does not exist. Add `@Index(value = ["last_message_at"])` on the entity itself.

`MessageEntity` — the `@ForeignKey` on `chatId` must specify `onDelete = ForeignKey.CASCADE`. Room requires a matching `@Index` on the FK column (`chatId`) for performance (lint will warn without it). Add both `Index("chat_id")` and `Index("created_at")` in the entity `indices` array.

**Foreign keys PRAGMA:**

Room does NOT enable FK enforcement by default. You must call `PRAGMA foreign_keys = ON` in a `RoomDatabase.Callback.onOpen`. Without this, `ON DELETE CASCADE` silently does nothing and `SanctumDatabaseTest::foreignKeyEnforcementEnabled` will fail. See the Room DAOs pattern in `patterns.md` for the standard `addCallback` approach used in this project.

**Schema export:**

The `room.schemaLocation` KSP argument must point to `$projectDir/schemas` (the `:app` module's `schemas/` subdirectory, not the repo root). After a successful `kspDebugKotlin`, the file appears at:
`app/schemas/app.sanctum.machina.data.SanctumDatabase/1.json`

This file must be committed to git as required by AC-R5. Add an exception to `.gitignore` if `app/schemas/*.local.json` is listed (the `.gitignore` only excludes `.local.json` variants, not `1.json`).

**Test setup:**

Instrumented tests in `app/src/androidTest/` need:
- `@RunWith(AndroidJUnit4::class)` on each test class
- `@get:Rule val db` built with `Room.inMemoryDatabaseBuilder(ApplicationProvider.getApplicationContext(), SanctumDatabase::class.java).allowMainThreadQueries().build()` — for test simplicity, `allowMainThreadQueries()` is acceptable
- `@After fun closeDb() { db.close() }`
- For Flow tests: use `db.observeAll().first()` via `runBlocking` or `runTest` — do not rely on `observeForever`
- For CASCADE test: the in-memory DB must also have PRAGMA foreign_keys = ON enabled (the Callback is called on open, so it fires automatically for in-memory DBs too)

**Edge cases:**

- `MessageEntity.text` has a NOT NULL DEFAULT '' constraint — when inserting a message with no text (e.g., audio-only), pass an empty string `""`, not null. The `@ColumnInfo(defaultValue = "")` annotation handles the schema; the Kotlin property type must be `String` (not `String?`).
- `chats.is_manually_titled` is stored as INTEGER (0/1) — use `@ColumnInfo(name = "is_manually_titled") val isManuallyTitled: Int = 0` rather than Boolean to avoid Room's Boolean-to-Int mapping quirks (Room handles Boolean automatically, but Integer is explicit and matches the SQL schema).
- Room 2.7.x and KSP: ensure `ksp` version in `libs.versions.toml` is compatible. Current `ksp = "2.3.6"` is compatible with `kotlin = "2.2.0"` and Room 2.7.x.

**Dependencies:**

- Task 4 (ChatRepository), Task 5 (WarmupCoordinator/AppModule), Task 6 (SanctumApplication rework) all depend on the entities and DAOs created here. Do not rename or restructure the public API without checking those tasks.
