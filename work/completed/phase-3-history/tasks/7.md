## Description

This task replaces the old `model_manager` start destination with `home` in `SanctumApp.kt` and wires up the full new navigation graph for Phase 3. It creates `HomeScreen` — the new entry point the user sees on every cold start — and `HomeViewModel` that drives its state.

**Why this task matters:** Phase 2 launched directly into Model Manager. Phase 3 needs a home hub matching the ChatGPT/Claude mental model: centered product identity, a single primary action ("Новый быстрый чат"), and a no-models placeholder for first-run. The `ModalNavigationDrawer` wrapping the `NavHost` makes persistent chat history permanently accessible via hamburger or swipe-right from anywhere in the app.

**Navigation changes:**
- Start destination: `model_manager` → `home`
- Retired route: `chat/{modelName}` (old string-based, used `Uri.encode/decode`)
- New routes added:
  - `chat/quick` — optional query arg `?modelId={String}` (for Model Manager "Load" flow, Task 11)
  - `chat/draft` — no args, persistent-chat draft mode before first Send
  - `chat/{chatId}` — `Long` nav arg, opens persisted chat by Room id
- `ModalNavigationDrawer` wraps the `NavHost`; `DrawerState` + `rememberCoroutineScope` live in `SanctumApp`; the hamburger icon callback is passed down via lambda to each screen's TopAppBar

**HomeScreen layout** (per `sanctum-screen-home.jsx` design):
- TopAppBar: hamburger (menu icon) left, "Sanctum · Home" label center, settings icon right
- Center-aligned body: `SmSigil` (Task 0), product name "Sanctum Machina", kicker line
- Primary action: `FilledTonalButton` "Новый быстрый чат" → navigates to `chat/quick`
- Secondary action: `TextButton` / outlined button "Открыть историю" → opens drawer (or navigates to drawer-content first visible item)
- Bottom status bar: warmup state dot + model chip (observes `ModelRegistry.activeModelName`)
- **No-models placeholder** (AC-F2, US-8): when `HomeViewModel.hasDownloadedModels == false`, replace center body with placeholder text "Для начала работы скачайте модель." and an "Открыть Model Manager" `TextButton`

**HomeViewModel** provides a single `StateFlow<Boolean>` — `hasDownloadedModels` — derived from `ModelRegistry.models` by checking whether any `ModelEntry` has `downloadStatus.status == ModelDownloadStatusType.SUCCEEDED`.

**ModelManagerScreen entry point** is moved to drawer footer navigation (also accessible from HomeScreen placeholder "Открыть Model Manager"). The `onLoad` callback from `ModelManagerScreen` now navigates to `chat/quick?modelId={modelId}` — but this full wiring is completed in Task 11. For now, the route `model_manager` stays registered in the `NavHost` (just no longer the start destination) so Task 11 can wire the new nav event without a merge conflict.

This task depends on Task 6 (`SanctumApplication` rework + `AppCorruptionState` created) because `HomeScreen` will display the corruption banner in Task 10 — but the banner wiring itself is Task 10's responsibility. Task 7 only creates the `HomeScreen` scaffold and `ModalNavigationDrawer`.
