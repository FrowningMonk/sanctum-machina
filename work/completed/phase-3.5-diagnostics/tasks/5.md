# Task 5: Pre-flight gate logic + Model Manager UI

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализовать pre-flight гейт скачивания модели в Model Manager: pure-функции `gateAllowsDownload(...)` и `formatRamShortage(...)`, типы `GateDecision` / `ModelRowState`, и UI-обвязку, которая блокирует кнопку «Скачать» с понятным русскоязычным объяснением на устройствах, не дотягивающих до `Model.minDeviceMemoryInGb`.

Это слайс №1 из tech-spec (Solution slice 1) — источник правды по гейту: `ModelManagerViewModel` подписывается на `registry.models`, комбинирует с `DeviceInfoProvider.totalMemoryBytes()` (читается один раз при старте VM, `totalMem` в runtime не меняется) и экспонирует `StateFlow<List<ModelRowState>>`. `ModelStatusSection` ветка `NOT_DOWNLOADED` принимает `gate: GateDecision` и при `!gate.allowed` рисует disabled-`Button` + `Text` с secondary-text. `onDownload` короткозамыкается при `!gate.allowed` (defence-in-depth — UI уже disabled).

Зависит от Task 1 (`AllowedModel.toModel()` маппит `minDeviceMemoryInGb` в `Model`; парсер отвергает `null` / out-of-range; рекалибровка JSON E2B 8→4, E4B 12→6). Без Task 1 поле приходит `null` и гейт всегда `false`.

Поведение sub-threshold устройства (Honor 200 в дефолтном allowlist'е проходит — обе модели `enabled`): для локальной верификации временно подменить порог в `model_allowlist.json` (например, E4B → 99) и пересобрать debug APK. Полный device matrix (S20 FE 5.3 ГБ, SM-G780F, realme RMX3085) собирается в Task 12 (Pre-deploy QA) — соответствует проектной памятке «Verify UI chain before device smoke».

**Edge cases:**
- `minDeviceMemoryInGb = null` после Task 1 не должно встречаться (parser отвергает), но `gateAllowsDownload` всё равно возвращает `false` в этом случае (defence-in-depth, AC).

- `formatGbFloor(0L)` → `"0.0"`; `formatGbFloor(1_073_741_823L)` → `"0.9"`; floor-arithmetic protected `Long.toLong() * 10L` от overflow до 9223 ГБ.
- Compose recomposition: `rows: StateFlow` collected как `State` — изменения registry triggerят recomposition, gate пересчитывается per-row. `totalBytes` — захваченная константа, recomposition не пересчитывает (правильно).

**Implementation hints:**
- Pure-функции в `GateDecision.kt` — top-level fun, без класса-обёртки. Тесты импортируют `import app.sanctum.machina.ui.modelmanager.gateAllowsDownload`.
- `formatRamShortage` строит строку конкатенацией (`"Недостаточно RAM (${formatGbFloor(totalBytes)} ГБ устройство, нужно $minGb ГБ)"`), без `String.format` (default-Locale risk).
- `formatGbFloor`: `(bytes * 10L) / 1_073_741_824L` — целочисленная арифметика, floor-by-default, не round-up. `Long`-литерал `10L` обязателен (иначе Int-overflow на больших байтах).
- `formatGbFloor` visibility: `internal` keeps the helper module-private (`:app`-only); other Gradle modules cannot see it — that's still safe, и при этом Task 8 (тот же `:app`) импортирует его кросс-файлово.
