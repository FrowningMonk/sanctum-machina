## Description

Задача завершает все UX affordances ChatScreen и HomeScreen, заявленные в user-spec Phase 3.

**ChatScreen — TopAppBar state machine (AC-U6, AC-U7, AC-E3, AC-E3b).**
TopAppBar в `ChatScreen.kt` сегодня — простой `Text(modelName)`. В Phase 3 он становится state machine с четырьмя режимами, управляемыми из `ChatViewModel` через новый `StateFlow<TopAppBarState>`:

1. **Draft** (`ChatIdentity.Draft`) — имя текущей модели показано как кликабельная кнопка `Button` / `DropdownMenu`-trigger с иконкой `chevronDown`. Тап → `DropdownMenu` со списком downloaded-моделей из `registry.models`. Выбор модели, не совпадающей с прогретой → `AlertDialog` "Переключить модель? Займёт 20–30 сек." → confirm → `warmupCoordinator.cancelAndRestart(selectedModelId)`. После первого Send draft становится Persistent и TopAppBar переходит в состояние Ready или Loading.

2. **Failed/Idle** (`ChatIdentity.Persistent`, статус `ModelInitStatus.Failed` или `Idle`) — кнопка «Загрузить» с иконкой `warn` и красной границей, как в дизайн-спеке `sanctum-screen-more.jsx` (state `failed`). Тап → `warmupCoordinator.cancelAndRestart(chat.modelId)`.

3. **Initializing** (любой identity, `WarmupCoordinator` в полёте) — маленький `CircularProgressIndicator` + текст "Модель перезагружается…". `Send` disabled. Дизайн: `sanctum-screen-more.jsx` state `loading`.

4. **Ready** (`ChatIdentity.Persistent`, статус `ModelInitStatus.Ready`) — read-only `Text` с именем модели, точка-статус `SmDot(state="ready")`. Не кликается.

**Incognito indicator для Quick mode (AC-U5).**
Когда `ChatIdentity == Quick`, в TopAppBar: иконка `SanctumIcons.EyeOff` (перечёркнутый глаз) рядом с именем модели + subtitle "Быстрый чат". Применяется `SanctumIncognitoTheme` (из Task 0) как обёртка вокруг всего экрана. Дизайн: `sanctum-screen-quick.jsx`.

**imePadding на MultimodalInputBar (AC-U4, US-11).**
В `ReadyContent` контейнер `MultimodalInputBar` (сейчас `Modifier.fillMaxWidth().padding(...)`) получает `Modifier.imePadding()`. Это поднимает кнопку Send над экранной клавиатурой без использования `WindowCompat.setDecorFitsSystemWindows` (уже настроен в `MainActivity`). Текущий код в `ChatScreen.kt` строки 273–278 — `MultimodalInputBar` без `imePadding`.

**Corruption banner на HomeScreen (AC-D5, US-14).**
`HomeScreen` (создан в Task 7) получает инъекцию `AppCorruptionState` (создан в Task 6) и `LogExportManager`. Если `AppCorruptionState.corruptionOccurred == true`, показывается одноразовый `Card`-баннер с текстом «Прошлая история оказалась повреждена. Старая база сохранена.» и двумя кнопками: «Сохранить лог» (вызывает SAF-экспорт через `LogExportManager.buildExport(ExportSource.About)` + `writeTo(uri, content)`) и «Закрыть» (скрывает баннер через `remember { mutableStateOf(false) }` — в памяти, не персистентно). После закрытия баннер не появляется до следующего запуска приложения.

Задача не трогает Room, WarmupCoordinator, навигацию и DrawerContent — всё это уже реализовано в Tasks 7 и 8. Задача только обновляет UI-слой двух экранов.

7. **Corruption banner в `HomeScreen`** — в параметры `HomeScreen` добавить `appCorruptionState: AppCorruptionState` и `logExportManager: LogExportManager` (получать через Hilt в `HomeViewModel` или передавать из `SanctumApp.kt`). Добавить `var showCorruptionBanner by remember { mutableStateOf(appCorruptionState.corruptionOccurred) }`. Баннер — `Card` с `warning`-тоном, текстом и двумя кнопками: «Сохранить лог» и «Закрыть».

8. **SAF launcher в `HomeScreen` для "Сохранить лог"** — использовать `rememberLauncherForActivityResult(ActivityResultContracts.CreateDocument("text/plain"))`, в `onResult` вызвать `logExportManager.buildExport(ExportSource.About)` + `writeTo(uri, content)` на `Dispatchers.IO` через `LaunchedEffect` / корутину.

### Edge cases

- **Corruption banner + SAF:** SAF launcher — один `rememberLauncherForActivityResult`. Не запускать второй пока первый не завершился (guard `var launching`), аналогично паттерну из `AboutScreen`.
- **Draft picker: только downloaded-модели.** В dropdown показываем только `ModelEntry` где `isDownloaded == true` (не `ModelInitStatus.Ready`). Поле `isDownloaded` отражает наличие файла на диске (Task 3 + registry scan), не runtime-состояние engine.
- **Cross-model dialog в Draft:** показывается только если выбранная модель ≠ `registry.activeModelName.value`. Если выбранная модель УЖЕ является активной (e.g. пользователь выбрал ту же) — без диалога, просто dismiss dropdown.
- **`isWarmupInProgress` StateFlow:** `WarmupCoordinator` должен экспортировать `val isWarmupInProgress: StateFlow<Boolean>`. Если этого поля нет — реализовать в Task 5 или добавить в этой задаче.
- **imePadding placement:** `Modifier.imePadding()` нужен именно на элементе, который должен подниматься над клавиатурой. В текущем `Scaffold`-паттерне правильное место — `Column` внутри `Scaffold` content lambda (строка 260 `ChatScreen.kt`), или непосредственно на контейнере `MultimodalInputBar`. Убедиться что `WindowCompat.setDecorFitsSystemWindows(window, false)` уже установлен в `MainActivity` — без этого `imePadding` не работает.
- **Incognito theme:** `SanctumIncognitoTheme` применяется как обёртка только для `ChatIdentity.Quick`. Не применять к loading/failed состояниям независимо от identity.
- **TopAppBar state при первом запуске Quick chat:** `ChatIdentity.Quick` + warmup ещё идёт → Loading state. Пользователь видит spinner, но incognito indicator всё равно виден в subtitle.

### Implementation hints

- `TopAppBarState.Draft` содержит `List<ModelEntry>` (downloaded models) и `currentModelId: String` — чтобы в dropdown пометить галочкой текущую модель.
- `ChatTopAppBarTitle` принимает `TopAppBarState` и соответствующие callback'и; это stateless composable, состояние dropdown (`expanded`) хранится внутри него через `remember`.
- Для «Загрузить» использовать `OutlinedButton` с `colors = ButtonDefaults.outlinedButtonColors(contentColor = MaterialTheme.colorScheme.error)` и `border = BorderStroke(1.dp, MaterialTheme.colorScheme.error)`.
- Текст «Модель перезагружается…» из tech-spec. Цвет: `MaterialTheme.colorScheme.onSurfaceVariant`.
- В design spec `sanctum-screen-more.jsx`: `SmStateBar.state="failed"` показывает `warn`-иконку + «Загрузить снова». В нашей реализации текст «Загрузить» (без «снова») — короче и по стилю UX guidelines.
- Corruption banner Card: использовать `MaterialTheme.colorScheme.errorContainer` как background, `onErrorContainer` как tint для текста.
- SAF filename format: `"sanctum-log-${SimpleDateFormat("yyyyMMdd-HHmm", Locale.ROOT).format(Date())}.txt"` — аналогично `AboutScreen`.
