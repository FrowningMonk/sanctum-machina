## Description

Задача закрывает два пользовательских flow в `ModelManagerScreen`: выбор модели по умолчанию и быстрый "try model" через quick chat.

**Что делаем:**

1. **Star-индикатор** — текущий default отображается с ведущей иконкой ⭐ в строке списка. Признак "текущий default" вычисляется из нового поля `AppSettings.default_model_id`, которое было добавлено в Task 2. `ModelManagerViewModel` получает новый `StateFlow<String>` с текущим `default_model_id`; composable сравнивает `entry.model.modelId` с этим значением.

2. **"Сделать по умолчанию"** — пункт в overflow-меню строки скачанной модели. При тапе: `modelManagerViewModel.setDefaultModel(modelId)` → `appSettings.setDefaultModelId(modelId)` → Snackbar "Модель по умолчанию: {name}". Пункт доступен только для скачанных моделей и скрыт для текущего default (нет смысла переставлять звезду туда, где она уже есть). Закрывает AC-F7 и US-8 (пункт 7).

3. **Routing-fix для "Загрузить"** — сейчас `onLoad(modelName)` эмитирует `NavEvent.OpenChat(modelName)`, а `SanctumApp.kt` навигирует на старый маршрут `chat/{modelName}`. В Phase 3 этот маршрут упразднён. Новый flow: `NavEvent.OpenQuickChat(modelId)` → `SanctumApp.kt` навигирует на `chat/quick?modelId={modelId}`. Закрывает AC-F4 и US-12.

4. **Инъекция `AppSettingsRepository`** в `ModelManagerViewModel` — добавить к существующим трём конструкторным параметрам (`ModelRegistry`, `CrashState`, `LogExportManager`).

Задача зависит от Task 7 (навигация — добавляет маршрут `chat/quick`) и Task 8 (`ChatViewModel` — обрабатывает `modelId` query arg для quick chat). Результат: полноценный UX выбора default и запуска quick chat из Model Manager.

Передать `onLoad` лямбду так, чтобы она принимала `modelId: String` (тип не меняется — `(String) -> Unit`, меняется только смысл строки).

5. **Убедиться что `Model.modelId` доступен** — поле `modelId: String` на `Model` появляется в Task 2. Если Task 2 ещё не влит, `ModelCard` не сможет сравнивать `entry.model.modelId`. - Добавить `val defaultModelId: StateFlow<String>` через `appSettings.observeDefaultModelId().stateIn(viewModelScope, SharingStarted.WhileSubscribed(5_000L), "")`. Метод `observeDefaultModelId()` добавлен в Task 2.

**`ModelManagerScreen.kt`** — текущее состояние: `LazyColumn` с `ModelCard`, `SnackbarHost` уже в Scaffold, `LaunchedEffect` обрабатывает `NavEvent.OpenChat`. Изменения:
- Добавить `val defaultModelId by viewModel.defaultModelId.collectAsStateWithLifecycle()`.
- В `LaunchedEffect` добавить ветки для `NavEvent.ShowSnackbar` (показывать через snackbarHostState) и `NavEvent.OpenQuickChat` (вызывать `onLoad(event.modelId)`). Убрать или переименовать ветку `NavEvent.OpenChat`.

### Edge cases

- Если `default_model_id` пустая строка (DataStore не содержит значения — первый запуск), `defaultModelId` StateFlow вернёт `""`, ни одна строка не получит ⭐. Это ожидаемое поведение (AC-F3: first model to be downloaded автоматически становится default — это логика WarmupCoordinator, не этой задачи).
- Если у модели нет `modelId` (строка пустая из-за незавершённой Task 2), сравнение `entry.model.modelId == defaultModelId` всегда будет `false` — звезда не появится, пункт в меню будет виден. Это безвредно.
- "Сделать по умолчанию" в overflow — только для `SUCCEEDED` status. Для `NOT_DOWNLOADED`, `IN_PROGRESS`, `FAILED` overflow-меню либо отсутствует, либо не содержит этого пункта.
- Overflow-меню должно закрываться после тапа на "Сделать по умолчанию" (стандартное поведение `DropdownMenuItem`).

### Implementation hints

- `DropdownMenu` в Compose требует `var expanded by remember { mutableStateOf(false) }` — объявлять внутри `ModelCard` (не `ModelStatusSection`), чтобы иметь доступ к иконке-триггеру в одном scope.
- Иконка `Icons.Filled.Star` доступна в `androidx.compose.material.icons:icons-extended`. Если не хочется добавлять heavy dep только ради одной иконки — использовать Unicode символ "⭐" в `Text` (но лучше иконку для accessibility + масштабирование).
- `SharingStarted.WhileSubscribed(5_000L)` для `defaultModelId` StateFlow — правило для ViewModel-exposed StateFlow per patterns.md.
- Не добавлять `ErrorLog.e(...)` в этой задаче — все ошибки DataStore уже покрыты `DefaultAppSettingsRepository` через `"settings-io"` компонент (Task 2).
