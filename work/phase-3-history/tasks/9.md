## Description

Реализуем `DrawerContent` composable и `DrawerViewModel` — полную UI-логику бокового drawer'а с историей persistent-чатов.

`DrawerViewModel` подписывается на `chatRepository.observeChats()`, сортирует чаты по `last_message_at DESC` и группирует по временным секциям: «Сегодня / Вчера / На этой неделе / Раньше». Также комбинирует список чатов с `registry.models` (StateFlow) чтобы определять доступность модели для каждого чата перед навигацией — это ключ к US-13: если файл модели удалён с устройства, drawer показывает диалог «Модель недоступна» вместо того чтобы открыть чат напрямую. Проверка доступности: `entry.downloadStatus.status == ModelDownloadStatusType.SUCCEEDED`.

`DrawerContent` отображает сгруппированный список. Каждая строка: название чата + chip с именем модели + относительная дата. Активный чат (совпадает с текущим `chatId` из nav back stack) выделен левой акцентной полосой `2dp`. Три интерактивных жеста на строке:
- **Swipe влево** — reveals красную кнопку «Удалить» → `AlertDialog` подтверждения (с названием чата и количеством сообщений) → `chatRepository.deleteChat()` + `popBackStack()` если удалённый чат был открыт.
- **Long-press** — `AlertDialog` «Переименовать» с pre-filled `TextField` (лимит 60 символов; пустой ввод сбрасывает auto-title, устанавливая `is_manually_titled = false`).
- **Tap** — pre-navigation check: если `entry.downloadStatus.status != ModelDownloadStatusType.SUCCEEDED` для `model_id` этого чата → диалог «Модель недоступна» с кнопками «Отмена» / «Скачать» (навигация в `model_manager`); иначе — навигация в `chat/{chatId}`.

Empty state: строка «Нет сохранённых чатов.» + кнопка «Новый чат».

Важное разграничение (Decision 7 из tech-spec): pre-navigation check использует `entry.downloadStatus.status == ModelDownloadStatusType.SUCCEEDED` (состояние файла на диске, из `registry.models`) — не `ModelInitStatus` (состояние runtime-движка). Это означает что чат с downloaded-но-не-прогретой моделью открывается нормально (read-first паттерн, AC-E3), а только по-настоящему отсутствующие файлы блокируются диалогом.

### Edge cases

- **Удаление открытого чата:** `deleteChat(chatId, currentOpenChatId)` — если совпадают, VM emit `DrawerEvent.PopBack` в `events: SharedFlow`. `DrawerContent` собирает flow через `LaunchedEffect` и вызывает `navController.popBackStack()`. VM не держит ссылку на `navController` (anti-pattern).
- **Стрим при удалении (US-7):** MVP-решение: Room CASCADE DELETE удаляет все сообщения; `messageDao.observeByChat(chatId)` начинает эмитить пустой список; `ChatViewModel` переходит в empty state. Никакой cross-VM сигнализации не требуется. Если удалённый чат был открыт — `DrawerEvent.PopBack` навигирует пользователя назад до того, как пустой список станет видимым. Note: если потребуется явная остановка inference-стрима до DELETE (например, в будущем), можно добавить `SharedFlow`-событие, которое `ChatViewModel` поймает и вызовет `stopResponse()` — но для MVP это излишне.
- **Пустой список чатов:** `DrawerUiState(sections = emptyList(), isLoading = false)` → показываем empty state.
- **`modelDisplayName` для chip:** брать из `ModelEntry.model.displayName` (или `Model.displayName` из allowlist). Если модель не найдена в registry (была удалена с allowlist) — показывать `model_id` как fallback.
- **Дата "На этой неделе":** строго `today - 7 days < date < yesterday`. Не включает сегодня и вчера.
- **Длинный заголовок чата:** обрезать через `Modifier` + `maxLines=1` + `overflow=TextOverflow.Ellipsis`.

### Implementation hints

- Проверка доступности модели: `entry.downloadStatus.status == ModelDownloadStatusType.SUCCEEDED`. Поле `downloadStatus` уже существует в текущем `ModelRegistry` — Task 3 не меняет эту структуру.
- Для группировки дат не нужен `Calendar` — `java.time.LocalDate` + `ChronoUnit.DAYS.between(date, today)` даёт чистую логику.
- `chatRepository.observeChats()` возвращает `Flow<List<ChatEntity>>` (или domain-объект). Уточнить тип по реальному интерфейсу из Task 4.
- Swipe в `LazyColumn` + `SwipeToDismissBox`: убедиться что `key = { it.id }` задан на `items(...)` для корректного recomposition при удалении строки.
- Hilt: `DrawerViewModel` в том же Gradle-модуле `:app` — дополнительная конфигурация модуля не нужна.
- Design tokens (Task 0): использовать `SanctumColors.accent` для левой полосы активного чата, `SanctumColors.error` для background swipe-reveal. Если Task 0 ещё не завершён, использовать `MaterialTheme.colorScheme.primary` / `error` как временный fallback.
