# Task 4: enableEdgeToEdge() in MainActivity and CrashReportActivity

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Фикс Bug 3 (визуальный зазор между поднятой клавиатурой и input bar на Honor 200 / EMUI). Корневая причина (см. code-research.md § 5): ни один экран приложения не вызывает `enableEdgeToEdge()` и не настраивает window insets вручную, поэтому Honor 200 не считает приложение edge-to-edge и оставляет под IME артефакт системного inset'а.

Решение по Decision 7: использовать единственную AndroidX-обёртку `enableEdgeToEdge()` из `androidx.activity:activity-compose` 1.10.1 (уже на classpath, `gradle/libs.versions.toml:8`). Никаких ручных `WindowCompat.setDecorFitsSystemWindows`, никакой ручной правки `statusBarColor` / `navigationBarColor` — они в коде и так нигде не выставлены, конфликтов нет.

Поскольку обе hosted-Activity (`MainActivity` для всех Compose-экранов и `CrashReportActivity` для recovery) — единственные точки входа окна, патч точечный: добавить вызов в `onCreate` ДО `setContent`. В `CrashReportActivity` вызов идёт ПОСЛЕ существующего `window.setFlags(FLAG_SECURE, FLAG_SECURE)` — это независимые window-API, порядок сохраняется, FLAG_SECURE не затрагивается (см. Decision 7 / Risks table).

Покрывает AC-3.1 (вызов есть в обеих Activity), AC-3.2 (нет зазора под IME), AC-3.3 (визуальный smoke на 6 экранах).

## What to do

1. В `app/src/main/kotlin/app/sanctum/machina/MainActivity.kt`:
   - Добавить импорт `androidx.activity.enableEdgeToEdge`.
   - В `onCreate` вызвать `enableEdgeToEdge()` сразу после `super.onCreate(savedInstanceState)` и ДО блока проверки `POST_NOTIFICATIONS` / `setContent { ... }`.
2. В `app/src/main/kotlin/app/sanctum/machina/crash/CrashReportActivity.kt`:
   - Добавить импорт `androidx.activity.enableEdgeToEdge`.
   - В `onCreate` вызвать `enableEdgeToEdge()` ПОСЛЕ существующей строки `window.setFlags(WindowManager.LayoutParams.FLAG_SECURE, WindowManager.LayoutParams.FLAG_SECURE)` и ДО `logExportManager = LogExportManager(...)` / `setContent { ... }`.
3. Не трогать ничего другого: ни `WindowCompat`, ни ручные `statusBarColor` / `navigationBarColor`, ни `fitsSystemWindows`. Никаких version-guard'ов (`Build.VERSION.SDK_INT >= ...`) — `minSdk = 31`, а API доступен с 21.
4. Убедиться, что lint и сборка проходят, и провести визуальный smoke на 6 экранах (AC-3.3).

## Details

**Files (текущее состояние и что сделать):**

- `app/src/main/kotlin/app/sanctum/machina/MainActivity.kt` (40 строк):
  - Сейчас: `class MainActivity : ComponentActivity()`, `onCreate` начинается с `super.onCreate(savedInstanceState)`, далее блок проверки `POST_NOTIFICATIONS` (Tiramisu+), затем `setContent { SanctumTheme { SanctumApp() } }`.
  - Изменения: добавить `import androidx.activity.enableEdgeToEdge` в блок импортов; в `onCreate` ВТОРОЙ строкой (сразу после `super.onCreate(savedInstanceState)`) вызвать `enableEdgeToEdge()`. Никаких других правок.

- `app/src/main/kotlin/app/sanctum/machina/crash/CrashReportActivity.kt` (~154 строки):
  - Сейчас: `onCreate` вызывает `super.onCreate(savedInstanceState)`, затем `window.setFlags(WindowManager.LayoutParams.FLAG_SECURE, WindowManager.LayoutParams.FLAG_SECURE)`, затем `logExportManager = LogExportManager(applicationContext)`, затем `setContent { SanctumTheme { CrashReportScreen(...) } }`.
  - Изменения: добавить `import androidx.activity.enableEdgeToEdge`; в `onCreate` вставить `enableEdgeToEdge()` ПОСЛЕ строки `window.setFlags(...)` и ДО `logExportManager = ...`. Никаких других правок (FLAG_SECURE, Hilt-исключение, SAF-launcher, `launching` guard — всё остаётся как есть).

**Dependencies:**
- Нет внешних зависимостей. `androidx.activity:activity-compose` 1.10.1 уже на classpath (`gradle/libs.versions.toml:8`). `enableEdgeToEdge()` доступен с 1.8.0+, версия 1.10.1 удовлетворяет с большим запасом.
- Нет блокирующих task'ов; параллельно с Task 3 / Task 5 в Wave 3.

**Edge cases:**
- **FLAG_SECURE + edge-to-edge:** независимые window-API; `setFlags` ставит флаг WindowManager, `enableEdgeToEdge()` настраивает SystemBarStyle и `setDecorFitsSystemWindows(false)`. Порядок не важен функционально, но по Decision 7 кладём после FLAG_SECURE для лёгкого ревью.
- **Тёмная / светлая тема:** `enableEdgeToEdge()` сам подбирает `SystemBarStyle.auto(...)` для status / nav bar, ручная цветовая логика не нужна. AC-3.3 валидирует обе темы визуально.
- **Existing `ChatScreen.kt:329` комментарий про `setDecorFitsSystemWindows`:** комментарий остаётся валидным (см. Decision 7 «Alternatives considered»), его не трогаем.
- **`minSdk = 31`:** API уровень 21+, version-guard не нужен; добавлять `if (Build.VERSION.SDK_INT >= ...)` — запрещено по Decision 7.
- **Honor-lock запрет (memory `manifest_breadth_over_honor_lock`):** фикс должен работать на стандартном AOSP edge-to-edge — `enableEdgeToEdge()` именно стандартный AOSP API, требование выполнено.

**Implementation hints:**
- `enableEdgeToEdge()` — extension-функция на `ComponentActivity`, вызывается без аргументов. Дефолтные `SystemBarStyle.auto(...)` подходят под эту задачу.
- Не вызывать в `onResume` / `onStart` — только в `onCreate` до `setContent` (вызов до построения UI обеспечивает корректное применение insets к корню Compose-иерархии).
- Если ide подсветит deprecated на чём-то по соседству — это не наш scope; не править.
- После правок прогнать `./gradlew :app:lintDebug :app:assembleDebug` локально перед review.
