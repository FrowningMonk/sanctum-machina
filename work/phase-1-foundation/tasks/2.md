## Description

Развернуть Gradle-инфраструктуру PhoneWrap: корневой `settings.gradle.kts` с двумя модулями (`:app`, `:core-runtime`), root `build.gradle.kts` с плагинами `apply false`, version catalog `gradle/libs.versions.toml` со всеми пиннингованными версиями из Dependencies секции tech-spec, минимальные `build.gradle.kts` в `:app` и `:core-runtime`, Gradle wrapper 8.x. Цель — первая успешная сборка пустых модулей, чтобы Wave 1 (Task 3/4/5) мог параллельно заливать код в `:core-runtime` и `:app` на готовом каркасе.

Версии берутся из `architecture.md § Tech Stack` (authoritative по Decision T2), а не 1-в-1 из Gallery. Hilt подключается через KSP, не kapt (Decision T3). Gallery-конфиги (`gallery-source/Android/src/`) используются как референс — копируем структуру pluginManagement, вырезаем Firebase / oss-licenses / protobuf / camera / richtext / OAuth / DataStore / splashscreen. Корень проекта — `C:\AI-WORK\PhoneWrap\`, package root — `app.sanctum.machina.*` (Decision T1 в tech-spec).

- `gradlew`, `gradlew.bat` — wrapper-скрипты.

**Dependencies (по задачам):** Task 1 (git init, `.gitignore` с записями `build/`, `.gradle/`, `local.properties`).

**Edge cases:**
- Windows CRLF: `.gitattributes` из Task 1 должен оставить `gradlew` (LF) и `gradlew.bat` (CRLF) корректно — проверить `git diff` на скачанных wrapper-файлах.
- `local.properties` (с `sdk.dir`) НЕ коммитим — должно быть в `.gitignore` от Task 1. Если ещё нет — добавить.
- Если `compileSdk = 35` требует Android SDK platform 35 на dev-машине — зафиксировать в README (но README не scope Task 2).
- Версии Compose BOM `2026.03.00` (architecture.md) vs Gallery `2026.02.00` — используем architecture.md (T2). Убедиться, что артефакт `androidx.compose:compose-bom:2026.03.00` существует в Google Maven — если нет, откатываемся на следующую доступную 2026.x и фиксируем в decisions.md как отклонение.

**Implementation hints:**
- Gradle wrapper проще всего получить через `gradle wrapper --gradle-version 8.11.1` в свежей директории или скопировать `gradle/wrapper/gradle-wrapper.{jar,properties}` + `gradlew` / `gradlew.bat` из `gallery-source/Android/src/` (убедиться, что версия wrapper'а там ≥ 8.x — Gallery использует 8.x). Исходные файлы wrapper'а Gallery — зеркало референсного дистрибутива, это безопасно копировать 1-в-1.
- В `libs.versions.toml` для Compose plugin используй `kotlin-compose = { id = "org.jetbrains.kotlin.plugin.compose", version.ref = "kotlin" }` (встроен в Kotlin 2.2 — никаких `kotlinCompilerExtensionVersion`).
- Для `compose-bom` — `compose-bom = { group = "androidx.compose", name = "compose-bom", version.ref = "composeBom" }`, остальные Compose-либы БЕЗ `version.ref` (подтягиваются BOM'ом).
- Alias `androidx-lifecycle-process = { group = "androidx.lifecycle", name = "lifecycle-process", version.ref = "lifecycle" }` — обязателен в `[libraries]`, используется напрямую в `:core-runtime/build.gradle.kts` (см. What to do шаг 5) для `ProcessLifecycleOwner`, который Task 3 подставит вместо вырезанного `AppLifecycleProvider`.
- `material-icons-extended` версию пиннуем явно (`materialIconExtended=1.7.8`) — не через BOM, т.к. Risk T-R4 в tech-spec: Compose BOM 2026.03.00 может конфликтовать с 1.7.8. Оставляем как указано, при конфликте — diagnose на Task 8 (ModelManagerScreen).
- `jvmTarget = "11"` — и source/target compatibility 11. Это согласовано с Kotlin 2.2.0 + AGP 8.8.2.
- `freeCompilerArgs += "-Xcontext-receivers"` в обоих модулях — opt-in для `@ExperimentalApi`, как в Gallery.
- `buildFeatures { compose = true; buildConfig = true }` в `:app` — `buildConfig = true` обязателен для `buildConfigField` (AGP 8 отключает по умолчанию).
- Для первой успешной сборки `AndroidManifest.xml` в обоих модулях достаточно пустого `<manifest />` — AGP 8 больше не требует атрибута `package` (берётся из `namespace`).
- Task 3 далее заведёт код в `:core-runtime` и сломает сборку, если KSP Hilt или модульная граница настроены неверно — но на этом этапе достаточно, чтобы `./gradlew projects` и `help` проходили.
