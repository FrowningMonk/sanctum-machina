# Task 4: `AllowlistLoader` + `CoreRuntimeModule` skeleton + bundled JSON + единственный Phase 1 unit-тест

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализовать загрузку списка разрешённых моделей из bundled-ассета вместо runtime-фетча (Decision T7) **и** создать скелет Hilt-модуля `CoreRuntimeModule`, в который последующие задачи (Task 5 — `ErrorLog`, Task 6 — `DownloadRepository`/`LlmModelHelper`/`ModelRegistry`) будут добавлять `@Provides`. Task 4 — первая задача в Wave 1, которая нуждается в Hilt-инфраструктуре для `:core-runtime`, поэтому ownership скелета закреплён именно здесь (cross-task validator round 1, critical finding).

Задача делает четыре вещи:

1. **Создаёт скелет `CoreRuntimeModule`** `core-runtime/src/main/kotlin/app/sanctum/machina/core/di/CoreRuntimeModule.kt` — пустой `@Module @InstallIn(SingletonComponent::class) object CoreRuntimeModule { }`. Без `@Provides`-методов — `AllowlistLoader` будет предоставлен Hilt автоматически через `@Singleton class AllowlistLoader @Inject constructor(...)`. Task 5 и Task 6 доложат сюда свои `@Provides`, либо пойдут тем же путём constructor-injection.
2. **Создаёт bundled JSON** `core-runtime/src/main/assets/model_allowlist.json` — форк `gallery-source/model_allowlists/1_0_11.json`, обрезанный до двух моделей: Gemma-4-E2B-it и Gemma-4-E4B-it. Это единственный источник правды про модели в Phase 1 — runtime-фетча из сети нет.
3. **Создаёт Hilt-injectable `AllowlistLoader`** с `suspend fun load(): Result<List<Model>>`. Аннотации: `@Singleton class AllowlistLoader @Inject constructor(@ApplicationContext context: Context)` — этого достаточно для auto-binding, `@Provides` в `CoreRuntimeModule` НЕ нужен. Читает ассет через `Context.assets.open`, парсит Gson-ом в `ModelAllowlist` классы (скопированные в Task 3), маппит через `AllowedModel.toModel()`. После парсинга прогоняет schema-guard по TAC-15 — при любом нарушении возвращает `Result.failure` (не per-entry skip).
4. **Пишет единственный Phase 1 unit-тест** `AllowlistLoaderTest.kt` на plain JUnit4 (~30 LoC). Фикстура в `src/test/resources/model_allowlist_fixture.json` — точная копия продовского ассета. Тест закрывает D8-gate user-spec'а и защищает Decision T8: ассершен `defaultConfig.accelerators` с `"gpu"` первым токеном ловит случай, если allowlist флипнется на `"cpu,gpu"` и GPU-fallback path сломается молча.

Почему один тест — потому что Phase 1 это foundation; всё остальное покрывается smoke-проверками и reviewer-ами. См. user-spec § Тестирование D8.

## What to do

1. **Форкнуть allowlist JSON:**
   - Прочитать `gallery-source/model_allowlists/1_0_11.json`.
   - Оставить в массиве `models` только две записи: `Gemma-4-E2B-it` и `Gemma-4-E4B-it` (litert-community варианты).
   - Сохранить формат/поля 1-в-1 — `name`, `modelId`, `modelFile`, `commitHash`, `sizeInBytes`, `defaultConfig`, остальные поля как в оригинале.
   - Положить результат в `core-runtime/src/main/assets/model_allowlist.json`.

2. **Создать `AllowlistLoader`:**
   - Путь: `core-runtime/src/main/kotlin/app/sanctum/machina/core/registry/AllowlistLoader.kt`.
   - Hilt-injectable (конструктор с `@ApplicationContext`), `@Singleton`.
   - API: `suspend fun load(): Result<List<Model>>`. Выполнение на `Dispatchers.IO`.
   - Открыть ассет `model_allowlist.json`, распарсить Gson-ом в тип `ModelAllowlist` (см. `core/data/ModelAllowlist.kt` из Task 3).
   - Для каждой записи прогнать schema-guard (см. TAC-15 ниже). Любое нарушение → немедленный `Result.failure(IllegalStateException("<reason>"))`, целиком load фэйлится.
   - Если guard прошёл — смаппить через `AllowedModel.toModel()` и вернуть `Result.success(list)`.

3. **Schema guard (TAC-15) — после парсинга:**
   - `require(list.isNotEmpty())` иначе `Result.failure(IllegalStateException("Empty allowlist"))`.
   - Для каждой записи:
     - `require(modelId.startsWith("litert-community/"))` — иначе fail с указанием нарушающего modelId.
     - `require(sizeInBytes in 1..10_737_418_240L)` (≤ 10 GB) — иначе fail.
     - Вычислить итоговый download URL (как это делает `AllowedModel.toModel()` / helper); `require(url.startsWith("https://huggingface.co/"))` — иначе fail.

4. **Написать единственный unit-тест:**
   - Путь: `core-runtime/src/test/kotlin/app/sanctum/machina/core/registry/AllowlistLoaderTest.kt`.
   - Plain JUnit4, ~30 LoC, без Robolectric.
   - Фикстура `core-runtime/src/test/resources/model_allowlist_fixture.json` — точная копия `model_allowlist.json` из assets.
   - Тесты:
     - `loadFromFixture_returnsListOfTwoModels` — загрузить фикстуру (через прямой Gson-парсинг, без Android Context), прогнать через тот же код-маппинг/гуард, что и production path. `assertEquals(2, result.getOrThrow().size)`.
     - `loadFromFixture_allModelsHaveRequiredFields` — для каждой модели:
       - `name`, `modelId`, `modelFile`, `commitHash` — non-empty.
       - `sizeInBytes > 0`.
       - `defaultConfig != null`, `defaultConfig.topK != null`, `defaultConfig.temperature != null`.
       - `defaultConfig.accelerators != null`, первый токен (split by `,`, trim) == `"gpu"` — защита Decision T8.

5. **Рефакторить парсинг так, чтобы тест мог жить без Android Context** — либо вынести core-парсинг (InputStream → `Result<List<Model>>`) в internal-функцию/helper, которую вызывает и production-`load()`, и тест. Сам `AllowlistLoader` при этом остаётся инжектируемым.

**Dependencies:**
- Task 2 (Gradle skeleton, `:core-runtime` модуль существует, test-runner настроен).
- Task 3 создаёт `ModelAllowlist.kt` с типами `ModelAllowlist`/`AllowedModel`/`AllowedModelConfig` и `toModel()`. Task 4 в той же волне (Wave 1), поэтому координируйся: если Task 3 ещё не завершён — блокируйся до его готовности или согласуй последовательность внутри волны. Без этих типов Task 4 не скомпилируется.

**Edge cases:**
- Пустой массив `models` → `Result.failure(IllegalStateException("Empty allowlist"))`.
- `modelId` не начинается с `litert-community/` → fail.
- `sizeInBytes <= 0` или `> 10_737_418_240L` → fail.
- Итоговый download URL не `https://huggingface.co/...` → fail.
- Невалидный JSON / отсутствующий ассет → Gson бросит, `AllowlistLoader.load()` оборачивает в `Result.failure` (не даём исключению утечь).
- Фикстура рассинхронизирована с production-ассетом — тест всё равно должен пройти на текущей фикстуре, но при расхождении смоук `jq '.models | length'` выявит дрейф.

**Implementation hints:**
- Не протаскивай Android Context в тест — вынеси core-парсинг в функцию, принимающую `InputStream`. Production `load()` открывает ассет, тест открывает classpath-ресурс.
- Schema-guard делай через `require(...) { "reason" }` внутри `runCatching { ... }`, либо явным `if (!ok) return Result.failure(...)` — оба варианта допустимы; главное — не per-entry skip, а early fail.
- URL computation — используй ту же логику, что уже есть в `AllowedModel.toModel()` (или его helper); если там URL собирается из `modelId` + `modelFile` + `commitHash`, вызывай helper и проверяй результат одним `startsWith`.
- Фикстуру положи в `core-runtime/src/test/resources/` (не в `assets/`), читать из теста через `javaClass.getResourceAsStream("/model_allowlist_fixture.json")`.
- Тест не должен быть параметризованным или использовать вспомогательные фреймворки — plain JUnit4 `@Test`, две функции, минимум механики.
- Ассершен `accelerators` — `assertEquals("gpu", accelerators.split(",")[0].trim().lowercase())` или эквивалент. Это строчное, но важное: защита Decision T8.
