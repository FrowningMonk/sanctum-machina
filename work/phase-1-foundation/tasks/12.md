# Task 12: Security Audit

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:security-auditor` — [skills/security-auditor/SKILL.md](~/.claude/skills/security-auditor/SKILL.md)

## Description

Полный security-audit всей Phase 1 — прочитать все source-файлы, созданные или модифицированные в Phase 1, и применить OWASP Mobile Top 10 к нашему Android-приложению. Цель — убедиться, что мы не затащили в проект уязвимостей, скрытых секретов или унаследованных слабостей из исходного Google AI Edge Gallery, а также что все TAC-гарантии (TAC-4, TAC-12, TAC-13, TAC-14) действительно выполняются на уровне кода и манифеста.

Задача выполняется в Audit Wave последовательно после Task 10 (Manifest & App Wiring), потому что финальные манифест-хардeнинги (`usesCleartextTraffic=false`, `allowBackup=false`, `fullBackupContent=false`, объявление foreground service type) делаются именно там, а без них security-audit не имеет финальной картины.

Отчёт ревьюера пишется в JSON. Задача не пишет кода, не фиксит найденное — только фиксирует findings. Фиксы (если нужны) — отдельной итерацией по указанию пользователя.

## What to do

1. Прочитать all Phase 1 source:
   - `app/src/main/AndroidManifest.xml`
   - весь `app/src/main/kotlin/app/sanctum/machina/**`
   - весь `core-runtime/src/main/kotlin/app/sanctum/machina/core/**`
   - весь `core-runtime/src/test/kotlin/**`
   - все `build.gradle.kts` (root, `:app`, `:core-runtime`)
   - `gradle/libs.versions.toml`
   - `.gitignore`
   - `core-runtime/src/main/assets/model_allowlist.json` (bundled)

2. Применить OWASP Mobile Top 10:
   - **M1 Improper Platform Usage** — permission surface (`INTERNET`, `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_DATA_SYNC`, `POST_NOTIFICATIONS`, `WAKE_LOCK`): минимально достаточны, ничего лишнего. Runtime-permission flow для `POST_NOTIFICATIONS` (Android 13+) корректен. Foreground service type объявлен в манифесте и совпадает с `startForeground(type=…)`.
   - **M2 Insecure Data Storage** — логи пишутся в `context.filesDir` (приватно), модели в `context.getExternalFilesDir(null)` (доступно adb shell, но PII там нет). В Phase 1 нет Room/DataStore/SharedPreferences с секретами.
   - **M3 Insecure Communication** — `usesCleartextTraffic="false"` в `<application>`. `AllowlistLoader` валидирует https-only URL + префикс `litert-community/` + bounded size (TAC-15).
   - **M4/M7 Data & Code Integrity** — sha256-проверка скачанного `.litertlm` в Phase 1 НЕ делается (user-spec R5 явно принимает этот риск). Минимум: bundled `core-runtime/src/main/assets/model_allowlist.json` содержит непустой `commitHash` для каждой записи, `expectedSize` в разумных границах.
   - **M5 Cryptography** — N/A (в Phase 1 криптография не используется).
   - **M8 Code Tampering** — `Class.forName` в `DownloadWorker` защищён package-prefix guard'ом, предотвращающим загрузку произвольных классов (TAC-14).
   - **Log content sanitization** — `ErrorLog` обрезает до 500 символов, вырезает переводы строк; в логах нет подстрок `Authorization`, `Bearer`, `token=`.

3. Специфические верификации:
   1. **TAC-13 (HF-token plumbing removed):** grep по `:core-runtime` и `:app` на `hfToken`, `HF_TOKEN`, `accessToken`, `huggingface.*token`, `Authorization.*Bearer` — должно быть 0 hits в нашем коде (исключая комментарии «не используем»).
   2. **TAC-14 (package-prefix guard):** в `DownloadWorker` есть явная проверка префикса имени класса перед `Class.forName`, падение при несоответствии.
   3. **No secrets in git:** grep по всему репо на hardcoded HF tokens (`hf_[A-Za-z0-9]{30,}`), API keys, private keys (`BEGIN.*PRIVATE KEY`) → 0 hits.
   4. **TAC-12 (manifest hardening):** в `AndroidManifest.xml` `<application>` содержит `android:allowBackup="false"`, `android:fullBackupContent="false"` (или ссылку на no-op xml), `android:usesCleartextTraffic="false"`.
   5. **Attack surface:** нет `WebView` в Phase 1, нет обработки `file://` схемы, нет untrusted deserialization кроме Gson на bundled asset (`core-runtime/src/main/assets/model_allowlist.json`).

4. Записать отчёт в `work/phase-1-foundation/logs/audit/security-review.json`:
   - верхнеуровневый verdict (`pass` / `pass-with-findings` / `fail`);
   - список findings (каждый: `severity` = info/low/medium/high/critical, `category` = OWASP-id или TAC-id, `location` = file:line, `issue`, `recommendation`);
   - отдельное поле `tacVerification` с булевыми полями `tac4`, `tac12`, `tac13`, `tac14`, `tac15`;
   - краткое summary на 3–5 строк.

## Details

**Files:**
- `work/phase-1-foundation/logs/audit/security-review.json` — создать (audit report, single output artefact задачи)

**Files to read (read-only):**
- `app/src/main/AndroidManifest.xml` — манифест, verify TAC-12 (allowBackup/fullBackupContent/usesCleartextTraffic), permission list, foreground service type
- `app/src/main/kotlin/app/sanctum/machina/**/*.kt` — весь app-модуль
- `core-runtime/src/main/kotlin/app/sanctum/machina/core/**/*.kt` — весь core-runtime
  - особенно: `AllowlistLoader.kt` (TAC-15 https + prefix + size), `DownloadWorker.kt` (TAC-14 package-prefix guard), `ErrorLog.kt` (sanitization)
- `core-runtime/src/test/kotlin/**` — единственный тест `AllowlistLoaderTest`
- `gradle/libs.versions.toml` — версии depиндов, flag на подозрительные legacy-версии
- `.gitignore` — проверить что `*.key`, `.env`, `credentials*`, `secrets/` игнорируются (или что их нет в репо)
- `core-runtime/src/main/assets/model_allowlist.json` — bundled trust anchor, проверить commitHash/size/url shape

**Dependencies:** Task 10 (manifest & wiring) должна быть завершена — без финализированного манифеста TAC-12 неверифицируем.

**Edge cases / что особо проверить:**
- Gallery-наследие: оригинальный AI Edge Gallery передавал HF token через WorkManager `Data.Builder().putString("hfToken", …)`; убедиться что в нашем `DownloadWorker` этого не осталось ни в inputData keys, ни в HTTP headers
- `Class.forName` и `Class.forName(...).newInstance()` — полный grep по `:core-runtime` и `:app`; каждое использование должно быть либо защищено prefix-guard'ом, либо документировано
- Logging helpers: grep на `Log.d/e/i/w/v` и `println` — проверить что через них не улетает URL со встроенным токеном, `Authorization` header, или stack trace с чувствительными данными
- Gson deserialization: `AllowlistLoader` — единственное место, куда допустим Gson в Phase 1; никакого Gson над сетевыми/user-provided JSON
- FileProvider / content:// / intent-filters: если есть — проверить `exported`, `grantUriPermissions`, и authority
- Network security config: если есть `res/xml/network_security_config.xml` — проверить что он не ослабляет `usesCleartextTraffic=false`

**Implementation hints:**
- Использовать Grep с `-n` и `glob` для локализации findings (file:line)
- Для каждого TAC явно процитировать соответствующий фрагмент (file:line) в отчёте
- Severity scale: `critical` — эксплуатируемая уязвимость; `high` — утечка секретов/PII; `medium` — ослабление defense-in-depth; `low` — best-practice deviation; `info` — observation
- Phase 1 не имеет крипто/auth/БД — ожидаемая картина: большинство OWASP-категорий N/A, фокус на M1/M2/M3/M8 и log sanitization
