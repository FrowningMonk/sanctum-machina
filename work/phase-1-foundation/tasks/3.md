# Task 3: Перенос + патч Gallery-ядра в `:core-runtime`

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Скопировать inference + download core из Google AI Edge Gallery (`gallery-source/Android/src/app/src/main/java/com/google/ai/edge/gallery/`) в модуль `:core-runtime` под новым package root `app.sanctum.machina.core.*` и применить серию ручных патчей — удалить HF-token плумбинг (Decision D3, user-spec AC-1), удалить Firebase Analytics вызовы (D3, AC-4), заменить Gallery-специфичный `AppLifecycleProvider` на `ProcessLifecycleOwner` (Decision T5), развязать `:core-runtime` от FQN `MainActivity` через runtime-bundle + package-prefix guard (Decision T6), и обрезать `ModelAllowlist.kt` до GPU/CPU-only логики (AICore/NPU убрать — user-spec D1).

Это база для Wave 2 (Task 6 — `DefaultModelRegistry`) и Wave 3 (Task 8 — `:app`-интеграция). Все патчи — точечные, на основе конкретных line-level находок из `work/phase-1-foundation/code-research.md` § 1, 2, 11. Никакого «творчества» не требуется: скопировать → применить диффы из спека → прогнать grep-ы → убедиться что `compileDebugKotlin` зелёный.

**Почему именно так (rationale):** задача механическая, но критически важная — от корректности переноса зависит вся Wave 2 (Task 6 `DefaultModelRegistry` вызывает `model.runtimeHelper` из `ModelHelperExt.kt`) и Wave 3 (`:app` инициализирует `DefaultDownloadRepository.mainActivityFqn`). Пропуск даже одного файла (например, `ModelHelperExt.kt`) или недоудалённый UI-импорт (например, `androidx.compose.ui.unit.dp` в `Consts.kt`) ломает либо компиляцию, либо модульную границу `:core-runtime` (TAC-3: zero `androidx.compose.*` imports).
