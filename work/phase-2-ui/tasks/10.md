# Task 10: MessageBubble extraction + ThinkingBlock + markdown rendering + ChatViewModel thinking accumulation + SYSTEM_PROMPT_DEFAULT ConfigKey + engine systemInstruction wiring

## Required Skills

Перед выполнением задачи загрузи:

## Description

Задача объединяет несколько связанных изменений, которые вместе формируют полную поддержку thinking-блоков и системного промпта в чате.

1. **MessageBubble extraction** — выносим приватный composable из `ChatScreen.kt` в отдельный файл `MessageBubble.kt` для чистоты архитектуры.
2. **ThinkingBlock composable** — новый collapsible компонент с визуальной левой линией (`drawBehind(outlineVariant)`), приглушённым текстом, `SafeMarkdown` внутри и авто-раскрытием пока идёт генерация.
3. **Message.thinkingText** — добавляем поле `thinkingText: String?` в data-класс `Message`.
4. **ChatViewModel thinking accumulation** — `resultListener` принимает 3-й аргумент (thinking chunk), накапливает в отдельном StringBuilder, пропускает если `!llmSupportThinking || !enableThinking`.
5. **awaitInitialize wiring (D24)** — `DefaultModelRegistry.awaitInitialize` пробрасывает `systemInstruction` в `engine.createConversation` на основе значения из `model.configValues[SYSTEM_PROMPT_DEFAULT.label]`. **ВАЖНО:** `ConfigKeys.SYSTEM_PROMPT_DEFAULT`, расширение `createLlmChatConfigs` параметром `defaultSystemPrompt`, и парсинг `defaultConfig.systemPromptDefault` из JSON уже сделаны в Task 1 (Wave 1). Task 10 только добавляет последнее звено — проброс значения в engine через `awaitInitialize`. Не дублировать.

**awaitInitialize wiring (D24) — key code:**
```kotlin
// DefaultModelRegistry.awaitInitialize — NEW systemInstruction plumbing
val systemPrompt = (model.configValues[ConfigKeys.SYSTEM_PROMPT_DEFAULT.label] as? String)
  ?.takeIf { it.isNotBlank() }
val systemInstruction: Contents? = systemPrompt?.let { Contents.of(listOf(Content.Text(it))) }
llmModelHelper.initialize(
  ...,
  systemInstruction = systemInstruction,   // was effectively null
  ...
)
```

**Edge cases:**
- `thinkingText` может быть `null` (модель не поддерживает thinking или флаг выключен) — ThinkingBlock не рендерится
- `systemPrompt` пустой или blank — `systemInstruction` должен быть `null`, не пустой объект
- Смена модели или `resetConversation` — thinking StringBuilder должен сбрасываться
- Collapsible state ThinkingBlock: `inProgress = true` → принудительно раскрыт, нельзя свернуть; после завершения — toggle доступен

**Implementation hints:**
- Левая линия ThinkingBlock: `Modifier.drawBehind { ... }` рисует линию цвета `MaterialTheme.colorScheme.outlineVariant`
- Мутный текст: `LocalContentAlpha` или `contentAlpha = 0.6f` / `color = MaterialTheme.colorScheme.onSurfaceVariant`
- `SafeMarkdown` — создаётся в Task 6 (зависимость: Task 10 → 9 → 8 → 7 → 6, поэтому доступен). Не существует до выполнения Task 6
- ThinkingBlock состояние: `var expanded by remember { mutableStateOf(inProgress) }` + `LaunchedEffect(inProgress) { if (inProgress) expanded = true }`
- resultListener 3-й аргумент — убедиться в сигнатуре колбека по исходнику gallery
