# Task 1: AC-1 — починить парсинг allowlist `llmSupport*` + `defaultConfig.systemPromptDefault`

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

В `AllowedModel.toModel()` не пробрасываются поля `llmSupportImage`, `llmSupportAudio`, `llmSupportThinking` из JSON в объект `Model`. Кроме того, из `defaultConfig` не парсится поле `systemPromptDefault`, которое нужно передавать в `createLlmChatConfigs(defaultSystemPrompt=...)`.

Это блокирует поддержку мультимодальных моделей и системного промпта по умолчанию (D24, D1 user-spec). Попутно нужно синхронизировать тестовую фикстуру с prod-файлом allowlist, иначе тест `fixtureMatchesProductionAsset` (TAC-6) будет падать.

## What to do

1. В `AllowedModel.toModel()` добавить проброс `llmSupportImage`, `llmSupportAudio`, `llmSupportThinking` из JSON в `Model`. Отсутствующие поля — `false` по умолчанию.
2. В `createLlmChatConfigs(...)` добавить параметр `defaultSystemPrompt: String = ""` и парсить его из `defaultConfig.systemPromptDefault` allowlist JSON (optional string, default `""`).
3. Расширить `AllowlistLoaderTest` — написать TDD-anchor тесты (см. раздел ниже) до реализации, убедиться что они падают, затем реализовать.
4. Синхронизировать `model_allowlist_fixture.json` с `model_allowlist.json` (fixture/prod sync).

## TDD Anchor

Тесты написать ДО реализации. Запустить → убедиться что падают → написать код → убедиться что проходят.

- `AllowlistLoaderTest::llmSupportImage_true_parsedCorrectly` — JSON с `llmSupportImage: true` → `Model.llmSupportImage == true`
- `AllowlistLoaderTest::llmSupportAudio_true_parsedCorrectly` — JSON с `llmSupportAudio: true` → `Model.llmSupportAudio == true`
- `AllowlistLoaderTest::llmSupportThinking_true_parsedCorrectly` — JSON с `llmSupportThinking: true` → `Model.llmSupportThinking == true`
- `AllowlistLoaderTest::llmSupport_missing_defaultsFalse` — поля отсутствуют в JSON → все три `false`
- `AllowlistLoaderTest::defaultConfig_systemPromptDefault_parsed` — `defaultConfig.systemPromptDefault: "You are helpful"` → передаётся в `createLlmChatConfigs`
- `AllowlistLoaderTest::defaultConfig_systemPromptDefault_missing_defaultsEmpty` — поле отсутствует → `defaultSystemPrompt == ""`
- `AllowlistLoaderTest::fixtureMatchesProductionAsset` — fixture и prod JSON идентичны по содержимому (TAC-6)

**Edge cases:**
- `llmSupport*` поля отсутствуют в JSON → default `false`, не падать с NPE/KeyError
- `defaultConfig` отсутствует целиком → `systemPromptDefault` = `""`
- `defaultConfig.systemPromptDefault` присутствует, но `null` → `""`
- fixture и prod могут иметь разный порядок ключей — сравнивать по содержимому, не строково

**Implementation hints:**
- Проект использует Gson — добавить поля `val llmSupportImage: Boolean = false` и т.д. в data class `AllowedModel` (Gson автоматически десериализует, missing → default). НЕ использовать `optBoolean` (это `org.json` API)
- `fixtureMatchesProductionAsset` тест уже существует — не дублировать, но убедиться, что fixture обновлён с новыми полями
- Сначала написать тесты, убедиться в падении, потом фиксить реализацию
