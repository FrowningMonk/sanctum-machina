# Task 9: AudioRecorderBottomSheet (AudioRecord, lifecycle-aware)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализуем bottom sheet для записи аудио через `AudioRecord` (raw PCM, 16 кГц, моно). Лист показывается по нажатию на mic IconButton в ChatScreen. Запись идёт в `Dispatchers.IO`, таймер считает секунды до 30, после чего запись останавливается автоматически. Кнопка «Остановить» сохраняет `ByteArray` в ViewModel как аудио-аттачмент с `durationMs`. Lifecycle-aware: при уходе в фон, во время звонка или блокировки экрана запись прерывается и лист закрывается (`LifecycleEventEffect(ON_PAUSE)`). `DisposableEffect` гарантирует освобождение `AudioRecord` при любом выходе из композиции. Mic-кнопка деактивирована, если аудио-аттачмент уже прикреплён. Ошибка инициализации `AudioRecord` логируется через `ErrorLog.e("audio", ...)` и отображается снекбаром.

Решения из tech-spec: D5 (AudioRecord raw PCM), D13 (mic disabled when audio attached), D14 (lifecycle-aware release), D27 (ErrorLog "audio"), R9 (учитывать MIUI/HarmonyOS совместимость).

**Edge cases:**
- `AudioRecord.state != STATE_INITIALIZED` (устройства MIUI/HarmonyOS — R9) → логировать и gracefully выходить
- Пользователь быстро открывает/закрывает лист до инициализации AudioRecord → dispose должен корректно работать
- `minBufferSize` может вернуть `ERROR` или `ERROR_BAD_VALUE` → обрабатывать как init failure
- Параллельный доступ к микрофону другим приложением → запись не инициализируется, показать ошибку

**Implementation hints:**
- `AudioRecord` создавать внутри `LaunchedEffect` или coroutine в `Dispatchers.IO`, не в Main-потоке
- Чтение буфера в цикле с `while (isRecording)`, накапливать в `ByteArrayOutputStream`
- Таймер через `LaunchedEffect` с `delay(1000)` в цикле
- Для permission использовать `rememberLauncherForActivityResult(ActivityResultContracts.RequestPermission())`
- `LifecycleEventEffect` из `androidx.lifecycle.compose`
- `ErrorLog.e` — проверить существующий паттерн логирования в проекте (D27)
