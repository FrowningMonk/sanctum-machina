# Task 7: Attachment + MultimodalInputBar + ThumbnailStrip + Photo Picker + ChatViewModel attachments state

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализуем поддержку медиавложений в чате. Добавляем sealed-иерархию `Attachment` для хранения изображений и аудио в памяти (решение D6). Компонент `MultimodalInputBar` заменяет базовое текстовое поле вводом с кнопками медиа (камера, галерея, микрофон), которые показываются условно по возможностям модели. Выбор фотографий — через Android Photo Picker с ограничением 10 штук (решение D7). Изображения автоматически масштабируются до ~1024×1024, чтобы избежать OOM (решение R5). `ThumbnailStrip` показывает миниатюры вложений с кнопкой удаления. `ChatViewModel` управляет состоянием вложений через `StateFlow`. `ChatScreen` интегрирует оба новых компонента.

## What to do

1. Создать `Attachment.kt` — sealed class с вариантами `Image(Bitmap)` и `Audio(ByteArray, Long)` в пакете `:app`.
2. Создать `MultimodalInputBar.kt` — Composable с `OutlinedTextField` + 3 `IconButton` (camera, gallery, mic, каждый условно виден по `model.llmSupportImage` / `model.llmSupportAudio`) + кнопка Send. Send задизейблен, если текст пустой и вложений нет.
3. Настроить Photo Picker через `ActivityResultContracts.PickMultipleVisualMedia(maxItems = 10)` внутри `MultimodalInputBar` или на уровне `ChatScreen`.
4. Создать `ThumbnailStrip.kt` — `LazyRow` миниатюр; каждая миниатюра — `Image(bitmap)` + `IconButton` ✕ для удаления.
5. Добавить в `ChatViewModel`:
   - `attachments: StateFlow<List<Attachment>>`
   - `addImages(uris: List<Uri>)` — декодирует каждый URI через `decodeSampledBitmapFromUri(1024, 1024)`; если суммарно >10 изображений — берёт первые 10 и показывает snackbar.
   - `addImageBitmap(bitmap: Bitmap)` — добавление bitmap напрямую (для CameraBottomSheet в Task 8, у камеры нет URI).
   - `removeAttachment(idx: Int)`
6. Обновить `ChatScreen.kt` — заменить/дополнить текущий input bar на `MultimodalInputBar`, добавить `ThumbnailStrip` над ним.
7. Добавить необходимые строки в `strings.xml` (сообщение snackbar о лимите, content descriptions для кнопок).

**Dependencies:** Tasks 5 (strings), 6 (deps/build config)

**Tech-spec decisions:**
- D6: вложения хранятся в памяти (in-memory), не на диске
- D7: Photo Picker (`ActivityResultContracts.PickMultipleVisualMedia`) вместо legacy `ACTION_PICK`
- R5: downscale до 1024×1024 через `BitmapFactory.Options` (inSampleSize) для предотвращения OOM

**Edge cases:**
- Пользователь выбирает 0 фото (отмена picker) — ничего не происходит
- Добавление фото когда уже 10 — snackbar, список не меняется
- URI недоступен (удалён, нет разрешений) — `decodeSampledBitmapFromUri` возвращает null, пропускаем с логом
- Очень большое изображение (>50 МП) — downscale должен не допустить OOM
- Модель не поддерживает изображения — кнопки gallery/camera скрыты, `addImages` не должен вызываться

**Implementation hints:**
- `decodeSampledBitmapFromUri` реализовать как extension-функцию `ContentResolver`, используя `BitmapFactory.decodeStream` с двухпроходным чтением (первый проход — только размеры через `inJustDecodeBounds = true`, второй — с вычисленным `inSampleSize`)
- Photo Picker требует `androidx.activity:activity-ktx` и `androidx.activity:activity-compose` (должны быть добавлены в Task 6)
- `PickMultipleVisualMedia(maxItems = 10)` — Android 13+ (API 33); на более старых версиях система автоматически фоллбекается на legacy picker, ограничение maxItems при этом может не работать — обрезка до 10 в `addImages` обязательна
- Thumbnail размер: 96×96 dp в `ThumbnailStrip`
