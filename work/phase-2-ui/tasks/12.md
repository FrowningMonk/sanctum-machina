# Task 12: Code Audit

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-reviewing` — [skills/code-reviewing/SKILL.md](~/.claude/skills/code-reviewing/SKILL.md)

## Description

Полный аудит качества кода Phase 2. Читаем все исходные файлы Phase 2 и проводим целостный ревью по следующим осям:

- **Module boundary** — `:core-runtime` и `:core-settings` не должны содержать Compose/Activity (TAC-7, TAC-8).
- **Lifecycle hygiene** — корректные bind/unbind CameraX, release AudioRecord.
- **Autoscroll performance** — нет лишних рекомпозиций при автоскролле.
- **Permission flow consistency** — единообразный паттерн запроса разрешений по всем фичам.
- **Markdown safety** — использование SafeMarkdown/SafeUriHandler везде, где рендерится пользовательский контент.
- **Cross-component duplicate init** — нет дублирующейся инициализации общих ресурсов.
- **Shared resources compliance** — соответствие таблице shared resources из tech-spec.

Результат — аудиторский отчёт.

## What to do

1. Собрать список всех исходных файлов Phase 2 (Kotlin/XML/Compose) из модулей, затронутых в tasks 1–11.
2. Прочитать каждый файл и зафиксировать наблюдения по каждой оси аудита.
3. Проверить границы модулей: grep по `import androidx.compose`, `import android.app.Activity` в `:core-runtime` и `:core-settings`.
4. Проверить lifecycle: каждый `CameraX.bindToLifecycle` имеет соответствующий unbind; каждый `AudioRecord` вызывает `release()` в `onCleared`/`onStop`.
5. Проверить autoscroll: убедиться, что `LazyListState.animateScrollToItem` вызывается только при реальном изменении данных, без side-effect в composition.
6. Проверить permission flow: единообразие паттерна (launcher → rationale dialog → settings deeplink) во всех фичах с разрешениями (камера, микрофон).
7. Проверить SafeMarkdown/SafeUriHandler: найти все точки рендеринга Markdown и убедиться в использовании безопасных компонентов.
8. Проверить дублирующуюся инициализацию: найти повторные `initialize()` вызовы для одних и тех же ресурсов в разных компонентах.
9. Сверить с таблицей shared resources из tech-spec: каждый ресурс инициализируется ровно одним владельцем.
10. Написать аудиторский отчёт — по одному разделу на каждую ось, с конкретными файлами и строками для каждой находки.
