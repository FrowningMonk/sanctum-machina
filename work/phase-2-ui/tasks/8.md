# Task 8: CameraBottomSheet (CameraX)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализуем `CameraBottomSheet` — модальный bottom sheet с живым превью камеры (CameraX `PreviewView`), кнопками «Снять» и «Закрыть». При нажатии «Снять» захватываем кадр через `ImageCapture.takePicture`, конвертируем `ImageProxy` в `Bitmap`, применяем поворот по EXIF-ориентации, передаём bitmap во ViewModel и закрываем шит. Запрашиваем разрешение CAMERA on-demand (не при старте). Отказ → snackbar; постоянный отказ → snackbar со ссылкой на настройки. `DisposableEffect` корректно привязывает и отвязывает `ProcessCameraProvider`. Ошибка инициализации CameraX → `ErrorLog.e("camera", ...)` + snackbar + dismiss.

Решения: D6 (in-memory bitmap), D11 (runtime permissions), D27 (ErrorLog "camera"), R8 (rotation lifecycle).

- CameraX libs: `androidx.camera:camera-core`, `camera-camera2`, `camera-lifecycle`, `camera-view`

**Edge cases:**
- Нет доступных камер на устройстве → обработать как ошибку инициализации
- Пользователь поворачивает устройство во время превью → `DisposableEffect` rebind + корректный rotation
- Захват занимает время → кнопка «Снять» должна быть disabled во время обработки (избегаем двойного нажатия)
- `imageProxy` должен быть закрыт (`imageProxy.close()`) в любом случае — успех или ошибка

**Implementation hints:**
- `ProcessCameraProvider.getInstance(context).addListener(...)` внутри `DisposableEffect` → в `onDispose` вызывать `provider.unbindAll()`
- `rotateBitmap`: использовать `Matrix().apply { postRotate(degrees.toFloat()) }` + `Bitmap.createBitmap`
- Для snackbar при постоянном отказе: `ActivityCompat.shouldShowRequestPermissionRationale` = false → кнопка «Настройки» с `Intent(Settings.ACTION_APPLICATION_DETAILS_SETTINGS)`
- `ErrorLog.e("camera", ...)` — следовать паттерну D27 из decisions.md
