# Task 3: Порт media utils (`MediaUtils`, `AudioClip`, `calculatePeakAmplitude`) в `:core-runtime/common/`

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Перенести вспомогательные медиа-утилиты из gallery `common/Utils.kt` в модуль `:core-runtime` в пакет `core/common/`. Цели:

1. `MediaUtils.kt` — функции декодирования изображений (`decodeSampledBitmapFromUri`, `rotateBitmap`, `calculateInSampleSize`) и вычисления амплитуды PCM-аудио (`calculatePeakAmplitude`). `calculateInSampleSize` выделяется как pure-функция (без Android-зависимостей), чтобы тестироваться в JVM без Robolectric.
2. `AudioClip.kt` — plain class (не data class) для хранения raw PCM-данных с sample rate (решение D5).
3. Расширить whitelist компонентов в `ErrorLog.kt` четырьмя новыми значениями (решение D27).
4. Добавить зависимость Robolectric 4.12 в `libs.versions.toml` и `core-runtime/build.gradle.kts` (решение D20).
5. Покрыть всё unit-тестами.

Зависит от Task 2 (оба модифицируют `libs.versions.toml`) — выполняется последовательно в Wave 2.

## What to do

1. Прочитать исходные файлы gallery: `gallery-source/Android/src/app/src/main/java/com/google/ai/edge/gallery/common/Utils.kt` и `Types.kt` — понять сигнатуры и логику функций.
2. Создать `MediaUtils.kt`: скопировать и адаптировать `decodeSampledBitmapFromUri`, `rotateBitmap`, `calculateInSampleSize`, `calculatePeakAmplitude`. `calculateInSampleSize` — pure функция (только примитивные типы, без `BitmapFactory.Options` как параметра).
3. Создать `AudioClip.kt`: plain class с полями `audioData: ByteArray` и `sampleRate: Int`.
4. Модифицировать `ErrorLog.kt`: (a) добавить runtime whitelist enforcement — сейчас whitelist задокументирован но НЕ проверяется в коде, нужно добавить `require(component in ALLOWED_COMPONENTS)` или аналог → `IllegalArgumentException` для неизвестных компонентов; (b) добавить новые компоненты `"attachment-decode"`, `"audio"`, `"camera"`, `"settings-io"` в whitelist; (c) изменить cause-chain bounding с текущих `MAX_FIELD_LEN = 500` на 200 символов (TAC-15).
5. Добавить в `libs.versions.toml`: `robolectric = "4.12"` и соответствующую библиотеку.
6. Добавить в `core-runtime/build.gradle.kts`: `testImplementation(libs.robolectric)`.
7. Написать TDD-тесты (до реализации): `MediaUtilsTest`, `AudioClipTest`, `ErrorLogTest`.
8. Добавить тестовый ресурс `test-image.jpg` для Robolectric-тестов.

**Tech-spec decisions:**
- D5 — AudioRecord raw PCM: `AudioClip` хранит сырые PCM-байты, не compressed audio
- D20 — Robolectric для Android API тестов (BitmapFactory, ExifInterface)
- D22 — MultimodalContentsBuilder ref (контекст для понимания, как MediaUtils будет использоваться)
- D27 — ErrorLog whitelist: четыре новых компонента

**Edge cases:**
- `calculatePeakAmplitude`: пустой буфер (длина 0) → возвращает 0 без деления на ноль
- `calculatePeakAmplitude`: нечётный размер буфера — обрабатывать корректно (ByteArray интерпретируется как Little-Endian shorts)
- **ВНИМАНИЕ:** gallery `calculatePeakAmplitude(buffer: ByteArray, bytesRead: Int): Int` имеет 2 параметра и возвращает `Int`. Redesign: можно оставить `Int` return type (tech-spec говорит «calculatePeakAmplitude» без уточнения return type). Если нужен `Float` — документировать отличие
- `AudioClip`: нечётный размер `audioData` не должен вызывать исключение в конструкторе
- `decodeSampledBitmapFromUri`: URI может указывать на несуществующий файл — обработать через Result/nullable

**Implementation hints:**
- `calculateInSampleSize(rawWidth: Int, rawHeight: Int, reqWidth: Int, reqHeight: Int): Int` — pure функция, принимает только Int. **ВНИМАНИЕ:** в gallery это `private fun calculateInSampleSize(options: BitmapFactory.Options, ...)` — нужен redesign в pure-функцию, а не copy-paste
- Robolectric тесты помечаются `@RunWith(RobolectricTestRunner::class)` и `@Config(sdk = [33])`
- pure JVM тесты (calculateInSampleSize, calculatePeakAmplitude) — обычные JUnit без аннотаций Robolectric
- В `libs.versions.toml` секция `[versions]`: `robolectric = "4.12"`, секция `[libraries]`: `robolectric = { group = "org.robolectric", name = "robolectric", version.ref = "robolectric" }`
