## Description

Подготовка базовой инфраструктуры приложения для Phase 2: строковые ресурсы, манифест с правами и privacy-хардингом, зависимости CameraX + compose-richtext + lifecycle-runtime-compose.

Задача решает три независимых, но логически связанных вещи:
1. **strings.xml** — переименование приложения и добавление всех Phase-2 строк (настройки, кнопки, ошибки, лейблы), чтобы в Compose-коде не было хардкода (TAC-11).
2. **AndroidManifest.xml** — объявление разрешений CAMERA и RECORD_AUDIO, `<uses-feature camera required="false">`, отключение бэкапа (`allowBackup=false`), привязка rules-файла (TAC-9, TAC-10, TAC-14, D11, D26).
3. **Зависимости** (D25, D8) — CameraX 1.4.2, compose-richtext (maven group `com.halilibo.compose-richtext`), lifecycle-runtime-compose в version catalog и `app/build.gradle.kts`.

## What to do

1. В `app/src/main/res/values/strings.xml`:
   - Переименовать `app_name` → `"Sanctum Machina"` (D18, AC-2)
   - Добавить строки для 7 settings-полей (системный промпт, температура, top-k, top-p, max-tokens, thinking-budget, timeout или аналоги по ux-guidelines)

   - Reinit progress (сообщение прогресса реинициализации)
   - Snackbar «Системный промпт применён»

2. В `app/src/main/AndroidManifest.xml` (модуль `:app`):
   - Добавить `<uses-permission android:name="android.permission.CAMERA"/>`
   - Добавить `<uses-permission android:name="android.permission.RECORD_AUDIO"/>`

   - Установить `android:allowBackup="false"` на теге `<application>`
   - Добавить `android:dataExtractionRules="@xml/data_extraction_rules"` на теге `<application>`

3. Создать `app/src/main/res/xml/data_extraction_rules.xml`:

4. В `gradle/libs.versions.toml`:
   - Добавить версию `camerax = "1.4.2"`
   - Добавить алиасы для CameraX (camera-core, camera-camera2, camera-lifecycle, camera-view)
   - Добавить версию compose-richtext и алиасы (maven group `com.halilibo.compose-richtext`)
   - Добавить алиас lifecycle-runtime-compose (если не существует)

5. В `app/build.gradle.kts`:
   - Добавить CameraX зависимости через libs-алиасы
   - Добавить compose-richtext зависимости через libs-алиасы
   - Добавить lifecycle-runtime-compose если отсутствует

**Files to modify:**
- `app/src/main/res/values/strings.xml` — переименовать `app_name`, добавить все Phase-2 строки
- `app/src/main/AndroidManifest.xml` — permissions, uses-feature, allowBackup, dataExtractionRules
- `app/build.gradle.kts` — добавить CameraX, compose-richtext, lifecycle-runtime-compose deps
- `gradle/libs.versions.toml` — добавить версии и алиасы для новых зависимостей

**Files to create:**
- `app/src/main/res/xml/data_extraction_rules.xml` — explicit exclude-all backup rules

**Tech-spec decisions:**
- D11 — permissions (CAMERA, RECORD_AUDIO, uses-feature required=false)
- D18 — app_name = "Sanctum Machina"
- D25 — compose-richtext deps
- D26 — privacy hardening (allowBackup=false, dataExtractionRules)
- D8 — compose-richtext maven group = `com.halilibo.compose-richtext`

**Dependencies:** задачи 2 и 3 должны быть выполнены (Wave 2 завершена перед стартом этой задачи)

**Edge cases:**
- `<uses-feature>` должен быть `required="false"` — устройства без камеры не должны быть исключены из Play Store
- `allowBackup="false"` может конфликтовать с дефолтным значением в манифесте форка — убедиться, что атрибут явно выставлен, а не просто отсутствует
- compose-richtext 1.0.0-alpha02 — альфа-версия, проверить совместимость с Compose BOM, используемым в проекте
- Строки для settings-полей уточнить по ux-guidelines.md — точные имена и тексты могут отличаться от placeholder-значений в задаче

**Implementation hints:**
- `data_extraction_rules.xml` формат: `<data-extraction-rules><cloud-backup><exclude .../></cloud-backup><device-transfer><exclude .../></device-transfer></data-extraction-rules>` с `domain="root"` для полного исключения
- Перед добавлением зависимости в `build.gradle.kts` убедиться, что алиас в `libs.versions.toml` уже объявлен (иначе — compile error)
- lifecycle-runtime-compose, скорее всего, уже есть в проекте (форк Google AI Edge Gallery) — проверить перед добавлением дубликата
