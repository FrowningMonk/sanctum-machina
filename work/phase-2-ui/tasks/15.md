# Task 15: Pre-deploy QA

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:pre-deploy-qa` — [skills/pre-deploy-qa/SKILL.md](~/.claude/skills/pre-deploy-qa/SKILL.md)

## Description

Финальное acceptance-тестирование перед деплоем. Задача запускает полный набор автоматических проверок (build, unit-тесты, lint, сборка APK), верифицирует TAC-1..TAC-15, строит таблицу маппинга AC-1..AC-22 (auto-verified vs user-verify-only) и генерирует `manual-smoke.md` — пользовательский чеклист для ручной проверки на реальном устройстве (Honor 200).

QA является финальным gate: задача 15 не завершена до явного подтверждения AC-22 пользователем.

## What to do

1. Запустить полный build: `./gradlew build`
2. Запустить тесты по модулям:
   - `./gradlew :core-runtime:test`
   - `./gradlew :core-settings:test`
   - `./gradlew :app:test`
3. Запустить lint: `./gradlew lintDebug`
4. Собрать debug APK: `./gradlew :app:assembleDebug`
5. Верифицировать TAC-6..TAC-15 (где автоматические команды не покрывают — задокументировать как auto-checked или deferred)
6. Построить таблицу маппинга AC-1..AC-22: какие критерии авто-верифицированы, какие требуют ручной проверки
7. Сгенерировать `work/phase-2-ui/manual-smoke.md`:
   - Чеклист US-1..US-7 с expected outcomes
   - Edge cases с ожидаемым поведением
   - Список deferred AC (AC-2, AC-13, AC-14, AC-16) с инструкциями для проверки на Honor 200
8. Зафиксировать итог: пройден / не пройден финальный gate AC-22

## Verification Steps

- `./gradlew :core-runtime:test` → all tests pass
- `./gradlew :core-settings:test` → all tests pass
- `./gradlew :app:test` → all tests pass
- `./gradlew lintDebug` → no Error-level issues
- `./gradlew :app:assembleDebug` → APK present in `app/build/outputs/apk/debug/`

### User

- Проверить `manual-smoke.md` чеклист на реальном устройстве Honor 200
- Верифицировать AC-2, AC-13, AC-14, AC-16 вручную согласно инструкциям в `manual-smoke.md`
- Пройти US-1..US-7 по сценариям из `manual-smoke.md`
- Дать финальное подтверждение AC-22

## Details

**Files:**
- `work/phase-2-ui/manual-smoke.md` — создать: пользовательский чеклист US-1..US-7, edge cases, deferred AC с инструкциями

**Dependencies:** задачи 12, 13, 14 (все аудиты) должны быть в статусе `done` до старта

**Edge cases:**
- Если какой-либо из auto-тестов не прошёл — задача не завершена, нужно фиксить в соответствующей задаче
- Если lint выдаёт Error-level — блокер; Warning-level — документировать, не блокер
- APK должен быть физически создан (проверить путь `app/build/outputs/apk/debug/`)
- TAC, которые невозможно проверить автоматически — явно маркировать как `deferred-to-user` в `manual-smoke.md`, не считать провалом

**Implementation hints:**
- `manual-smoke.md` должен быть пригоден для использования пользователем без помощи агента: конкретные шаги, expected outcomes, pass/fail критерии
- Таблица маппинга AC — отдельный раздел внутри `manual-smoke.md` или отдельный блок в отчёте задачи
- AC-22 закрывается только явным словом пользователя — не закрывать автоматически
