# Task 13: Security Audit

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:security-auditor` — [skills/security-auditor/SKILL.md](~/.claude/skills/security-auditor/SKILL.md)

## Description

Полный аудит безопасности фичи phase-2-ui по стандартам OWASP Top 10 и OWASP MASVS. Цель — выявить уязвимости до того, как фича уйдёт в main. Аудитор является конечной точкой ревью; отдельный ревьюер не нужен.

Покрываемые области: модель разрешений (on-demand vs startup), privacy-настройки AndroidManifest, OOM-векторы при работе с вложениями, обработка content URI, корректность DataStore, Intent-обработчики, схемы SafeUriHandler, отсутствие захардкоженных секретов.

## What to do

1. Применить чеклист OWASP Top 10 и OWASP MASVS ко всему коду фичи phase-2-ui.
2. Проверить модель разрешений: разрешения запрашиваются on-demand, а не при старте приложения.
3. Верифицировать privacy-настройки: `allowBackup=false` и `dataExtractionRules` активны в AndroidManifest.
4. Оценить OOM-векторы при загрузке вложений: обработка in-memory изображений и файлов.
5. Проверить обработку content URI в `decodeSampledBitmapFromUri` — корректность, утечки, path traversal.
6. Проверить DataStore: файл с приватными правами, нет world-readable.
7. Проверить Intent-обработчики (`ACTION_APPLICATION_DETAILS_SETTINGS` и другие): нет лишнего exposure.
8. Верифицировать `SafeUriHandler` — перечислить заблокированные схемы (intent, sms, tel, custom schemes), убедиться в полноте списка.
9. Проверить кодовую базу на наличие захардкоженных токенов, ключей, паролей.
10. Сформировать отчёт о безопасности с находками и рекомендациями.

## Acceptance Criteria

## Verification Steps

### Smoke

- Убедиться, что `AndroidManifest.xml` содержит `allowBackup="false"` и ссылку на `dataExtractionRules`.
- Убедиться, что `SafeUriHandler` содержит явный список заблокированных схем.
- Убедиться, что поиск по кодовой базе (`grep -r "api_key\|token\|secret\|password"`) не выдаёт захардкоженных значений.

### User

- Пользователь проверяет отчёт аудита и подтверждает, что все критические и высокоприоритетные находки зафиксированы.

## Details

**Files:**
- `AndroidManifest.xml` — allowBackup, dataExtractionRules, permissions, Intent filters
- `SafeUriHandler.kt` (или аналог) — схемы блокировки URI
- `ChatViewModel.kt` — attachment state (OOM-векторы, in-memory bitmap loading); `AttachmentViewModel.kt` не существует — attachment state живёт в `ChatViewModel`
- `decodeSampledBitmapFromUri` — content URI handling, path traversal
- Все файлы DataStore — права доступа к файлу, отсутствие world-readable
- `build.gradle` / `local.properties` — проверка на секреты
- Исходники проекта целиком — поиск захардкоженных токенов

**Dependencies:** Task 11 (должна быть завершена перед аудитом — аудит проводится по финальному коду)

**Edge cases:**
- Схемы URI с нестандартными регистрами (`Intent://`, `SMS:`)
- Bitmap loading с corrupt/oversized content URI
- DataStore при первом запуске (файл ещё не существует)
- Intent с дополнительными extras, которые могут leak данные

**Implementation hints:**
- Использовать OWASP MASVS v2 как чеклист
- Для поиска секретов — `gitleaks` или ручной grep по паттернам
- OOM-оценка: проверить наличие `BitmapFactory.Options.inSampleSize` и лимитов размера
- SafeUriHandler: сравнить список схем с реальными угрозами (deeplink hijacking, tel: exfiltration)
