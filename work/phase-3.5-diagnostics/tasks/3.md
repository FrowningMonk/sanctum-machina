# Task 3: LogcatReader argv `*:E` → `*:W`

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Расширить фильтр logcat-сборщика с уровня ERROR (`*:E`) до WARN (`*:W`), чтобы в экспортируемом `.txt` оказывались предсмертные WARN-записи LiteRT-LM (которые сегодня теряются между последним WARN и фактическим ERROR/SIGSEGV). Изменение точечное: один токен в фиксированном `argv` плюс синхронный апдейт пин-теста, который этот argv проверяет.

Обоснование риска `*:W`-флуда отдельно зафиксировано в tech-spec → Risks (`*:W` логкат флудит на noisy OEM): tail-truncation в `LogExportManager` (cap = `MAX_LOGCAT_BYTES = 100 * 1024` байт ≈ 670 строк threadtime) сохраняет самые свежие записи — то есть именно тот хвост, ради которого расширяем фильтр. Менять `MAX_LOGCAT_BYTES` в этой задаче НЕ нужно (AC-L3 верифицирует cap отдельно в Pre-deploy QA Task 12 вручную на Honor 200).

Эта задача закрывает AC-L1 (фильтр включает WARN) и сохраняет AC-L2 (никаких других изменений в семантике сбора — таймаут, плейсхолдеры, drain-протокол не трогаем).

## What to do

Других правок в этом файле не делать — drain-протокол, обработка `IOException`/`SecurityException`, `LOGCAT_TIMEOUT_MS`, плейсхолдеры (`[logcat unavailable: ...]`) остаются как есть.
2. В `app/src/test/kotlin/app/sanctum/machina/logexport/LogcatReaderTest.kt`, строка 102: синхронно обновить ассерт пин-теста `argvShape_exactlySixArgs_knownPositions` — `assertEquals("*:E", argv[5])` → `assertEquals("*:W", argv[5])`. Остальные ассерты (длина argv == 6, позиции 0-4, regex для `--pid=\d+`) не трогать.
3. Запустить `./gradlew :app:test --tests "*LogcatReaderTest*"` и убедиться, что все тесты зелёные (`argvShape_*` обновлённый, четыре сценария плейсхолдеров без изменений, happy-path).
4. `LogExportManager.kt` НЕ модифицировать — `MAX_LOGCAT_BYTES = 100 * 1024` остаётся прежним, tail-truncation strategy валидна.

## Details

**Files:**
- `app/src/main/kotlin/app/sanctum/machina/logexport/LogcatReader.kt` — текущее состояние: класс `LogcatReader @Inject constructor(runner: CommandRunner)` + secondary-ctor `()` для не-Hilt путей, метод `read()` строит `argv` из 6 элементов (`logcat`, `-d`, `-v`, `threadtime`, `--pid=${Process.myPid()}`, `*:E`), вызывает `runner.run(argv, LOGCAT_TIMEOUT_MS = 2_000L)`, маппит `LogcatResult` в строку или `[logcat unavailable: <reason>]`. Драйвер `DefaultCommandRunner` (drain-thread + `waitFor(timeout)` + destroy-on-timeout) и `data class LogcatResult` живут в этом же файле — их не трогать. Изменение: только токен на строке 37.
- `app/src/test/kotlin/app/sanctum/machina/logexport/LogcatReaderTest.kt` — текущее состояние: Robolectric (`@RunWith(RobolectricTestRunner::class) @Config(sdk = [33])`) — нужен потому, что `LogcatReader.read()` зовёт `Process.myPid()`. Шесть тестов через hand-rolled `StubCommandRunner` / `RecordingCommandRunner` (никакого Mockito/MockK). Изменение: одна строка ассерта в `argvShape_exactlySixArgs_knownPositions` — `argv[5]` ожидается `"*:W"` вместо `"*:E"`.

**Dependencies:** нет (wave 1, depends_on пуст). Не требует Task 4 (InitDiagnostics seam) или каких-либо других слайсов.

**Edge cases:**
- Семантика плейсхолдеров не меняется — `[logcat unavailable: empty]` всё ещё триггерится при пустом stdout (например, на OEM, где `logcat` для own-pid возвращает 0 строк под `*:W`-фильтром, как и под `*:E`). Тест `emptyStdout_returnsUnavailableEmpty` должен остаться зелёным без изменений.
- `--pid=${Process.myPid()}` ограничивает выдачу собственным процессом — это уже исключает большую часть OEM-флуда (логи системных сервисов не попадут).
- Cap 100 KB в `LogExportManager.truncateLogcat` сохраняет хвост — самые свежие WARN/ERROR. AC-L3 ловится в pre-deploy-qa, не здесь.

**Implementation hints:**
- Изменение тривиально (один токен). Главное — НЕ затронуть соседние строки в `argv`-листе (форматирование, порядок) и НЕ менять секундарные конструкторы / drain-логику.
- Перед фиксацией кода прогнать `./gradlew :app:test --tests "*LogcatReaderTest*"` локально — должно быть 6 passed.
- Никакой shell-обёртки вокруг argv — `ProcessBuilder(argv)` принимает `List<String>` напрямую, токен передаётся как отдельный элемент списка (Decision 8).
