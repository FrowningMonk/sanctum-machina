## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Добавить ~12 новых русскоязычных ключей в `app/src/main/res/values/strings.xml`. Эти строки покрывают все пользовательские поверхности фичи Phase 2.5 — экрана `CrashReportActivity` (заголовок, тело, две кнопки), Toast/Snackbar на путях успеха и ошибки SAF-записи, restart-баннера на `ModelManagerScreen` (текст + contentDescription для иконки закрытия), раздела «Диагностика» в `AboutScreen` и диалога dev-gesture для тест-краша.

Задача — чистая подготовка ресурсов (plumbing): сами Compose-экраны, которые будут читать эти ключи, создаются в задачах 4 (CrashReportActivity + RestartCrashBanner), 6 (AboutScreen) и 7 (ModelManagerScreen). Все три задачи Wave 2 требуют, чтобы ключи уже существовали — отсюда `wave: 1` и отсутствие зависимостей. Тон и формулировки — по `ux-guidelines.md § Tone of Voice` и `§ Text Patterns`: короткие фразы, императив для кнопок, без «пожалуйста» / эмодзи / восклицаний, точка в конце error-сообщений.

## TDD Anchor

Изменения в `strings.xml` не покрываются типовым Kotlin unit-тестом — ресурсы не имеют поведения, которое имело бы смысл проверять через JUnit/Robolectric отдельно от экранов-потребителей. Стратегия верификации:

- **Compile check:** `./gradlew :app:assembleDebug` — падает с понятной ошибкой AAPT при любой синтаксической поломке XML (незакрытый тег, дубликат `name`, некорректная escape-последовательность типа незащищённого `'` в строке).
- **Визуальная инспекция ключей в задачах-потребителях:** правильность формулировок и тон проверяются автоматически тем, что задачи 4 / 6 / 7 ссылаются на конкретные ключи через `stringResource(R.string.…)`. Недобавленный ключ приведёт к compile error (`Unresolved reference: …`), лишний/неиспользуемый — к `UnusedResources` lint-предупреждению в AC tech-spec.
- **Lint:** `./gradlew :app:lintDebug` — проверяет `Typos`, `MissingTranslation`, `UnusedResources`. Для Phase 2.5 русский — единственная локаль (`ux-guidelines.md § Localization`), поэтому `MissingTranslation` не должен срабатывать.

Полноценная runtime-проверка (как строки выглядят в `CrashReportActivity`, `AboutScreen`, баннере) происходит в задачах 4, 6, 7 — именно там вводится Verify-user для UI-поведения.

**Dependencies:** нет — задача чисто plumbing. Blocks Wave 2 (tasks 4, 6, 7).

**Edge cases:**
- Апостроф (`'`) или двойные кавычки (`"`) внутри строки — должны быть экранированы (`\'`, `\"`) или строка обёрнута. В текущих формулировках не ожидается, но проверить при финализации.
- Ampersand (`&`) — требует `&amp;`. В перечисленных формулировках отсутствует.
- Многострочные значения — не использовать; все строки однострочные.
- Случайный конфликт с существующим ключом (`btn_close`, `btn_cancel`, `app_name`) — AAPT выдаст ошибку «Resource entry ... is already defined». Превентивно свериться с текущим файлом.

**Implementation hints:**
- Придерживаться стиля `name="snake_case"` — весь файл уже в этой конвенции.
- Ключи одного контекста группировать подряд (`crash_report_title`, `crash_report_body`, `crash_report_save` или аналогично) — по аналогии с блоком `Inference settings`.
- Точку в конце полных предложений (текст баннера, возможно тело экрана) ставить; короткие Toast/кнопки — без точки (формат из файла: `btn_download`, `settings_semilight_applied_snackbar`).
- При переиспользовании `btn_close`/`btn_cancel` в новой секции **не делать** алиасов (`crash_report_close` → `@string/btn_close`) — Compose-код задач 4/6/7 просто сошлётся на `R.string.btn_close` напрямую. В отчёте задачи зафиксировать: «используем btn_close для CrashReportActivity; новый ключ не добавляем».
- Для баннера: рассмотреть два варианта именования — `crash_banner_body` или `restart_banner_body`. Выбрать тот, что точнее описывает контекст; в tech-spec используется термин `RestartCrashBanner` — логично `restart_crash_banner_*` или компактнее `crash_banner_*`.

**Tone sanity-check (перед коммитом):**
- Нет «пожалуйста», «к сожалению», «упс», «ой».
- Нет эмодзи.
- Нет восклицательных знаков.
- Кнопки — императив.
- Error — одно предложение, без ретрай-хинта (кнопки ретрая в UI этой фичи нет — пользователь перезапускает SAF вручную).
